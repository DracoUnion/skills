# abcd-model

## When to Use

当需要快速识别和运用四种基础认知关系（并列、包含、因果、辩证）来结构化思考或解决问题时description

## Core Logic

# 执行步骤

1.  **识别关系类型**
    *   **输入**：待分析的问题、信息或一组元素。
    *   **操作**：判断这些元素之间最核心的关系属于以下哪种类型：
        *   **A (And/并列)**：元素之间是同级、相邻、重叠或互补关系。
        *   **B (Belong/包含)**：元素之间存在整体与部分、上层与下层的层级关系。
        *   **C (Cause/因果)**：元素之间存在时间先后、逻辑推导或前因后果的链条关系。
        *   **D (Dialectical/辩证)**：元素之间相互依存、相互转化、对立统一。
    *   **判断标准**：能够清晰地将问题或信息映射到一种关系类型上。
    *   **输出**：确定的关系类型（A/B/C/D）。

2.  **应用对应模型框架**
    *   **输入**：步骤1确定的关系类型。
    *   **操作**：根据关系类型，调用相应的思维工具或框架进行结构化分析：
        *   **IF 关系类型为 A (并列)**：使用地图、矩阵（如波士顿矩阵）、维恩图或公式来界定和展现元素间的相邻、重叠或互补关系。
        *   **IF 关系类型为 B (包含)**：使用金字塔结构、树状图或层级列表来展现整体与部分、归纳与演绎的层级关系。
        *   **IF 关系类型为 C (因果)**：使用流程图、时间线、因果链或操作指南来梳理事件的先后顺序、发展趋势和前因后果。
        *   **IF 关系类型为 D (辩证)**：使用平衡理论模型（如海德平衡三角）、阴阳图或相生相克图来展现元素间的相互依存、对立与转化关系。
    *   **判断标准**：选用的工具能够有效、清晰地呈现步骤1所识别的关系。
    *   **输出**：一个初步的结构化分析框架或图形化表示。

3.  **提炼核心洞察与行动**
    *   **输入**：步骤2生成的结构化分析结果。
    *   **操作**：基于结构化框架，提炼核心矛盾、关键杠杆点或可行方案。例如：
        *   对于A类关系，明确边界或寻找组合机会。
        *   对于B类关系，定位问题层级或确定行动优先级。
        *   对于C类关系，识别根本原因或预测后续发展。
        *   对于D类关系，寻找对立面的统一路径或促进良性转化。
    *   **判断标准**：得出的洞察直接指向问题的解决或决策的优化。
    *   **输出**：明确的结论、决策建议或下一步行动计划。

# 输出格式要求

以结构化方式呈现分析结果：
1.  **识别的关系类型**：[A/B/C/D] - [关系类型中文名称]
2.  **应用的框架/工具**：[具体工具名称，如波士顿矩阵、金字塔原理等]
3.  **分析图示/结构**：（可选）用文字描述或ASCII字符简单勾勒分析结构。
4.  **核心洞察**：[基于模型分析得出的关键结论]
5.  **行动建议**：[具体的、可执行的下一步建议]
