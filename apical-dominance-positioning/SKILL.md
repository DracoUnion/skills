# apical-dominance-positioning

## When to Use

当需要在竞争激烈的领域或赛道中建立或评估长期竞争优势时description

## Core Logic

# 执行步骤

1.  **定义竞争格局与评估本质**
    *   **输入**：对市场、行业、机会集合或合作关系的初步理解。
    *   **操作**：
        *   **IF** 目标是建立竞争优势或筛选机会：清晰界定你所关注的竞争范围（“赛道”）。
        *   **IF** 目标是构建生态或深化合作：分析当前领域的价值创造与分配逻辑，判断竞争是零和博弈还是存在生态共赢潜力。
    *   **判断标准**：赛道边界明确，或能清晰描述领域内的竞争/合作本质。
    *   **输出**：明确的“赛道”定义，或关于竞争本质（零和/生态）及自身潜在角色（参赛者/赋能者）的初步判断。
    *   **分支逻辑**：
        *   **IF** 判断为存在显著生态扩张潜力，且自身具备成为赋能者的可能 → 进入**路径A：生态构建者**。
        *   **ELSE** (目标为传统竞争、机会筛选或深化双边合作) → 进入步骤2。

2.  **识别顶端位置与关键节点**
    *   **输入**：明确的赛道定义或机会集合。
    *   **操作**：
        *   **IF** 分析整个赛道：收集并分析关键参与者的数据（市场份额、增长率、品牌影响力等），识别1-3个在关键指标上显著领先的“顶端”玩家。
        *   **IF** 筛选高潜力机会：应用“三层过滤法”：
            a.  **第一层**：聚焦20%处于上升趋势的“顶端行业”。
            b.  **第二层**：在每个顶端行业内，聚焦20%具有领先优势的“顶端企业/模式”。
            c.  **第三层**：锁定1-3个最终目标，准备集中投入。
    *   **判断标准**：能够基于数据识别出顶端玩家，或通过过滤法将机会范围大幅收敛至极少数高潜力选项。
    *   **输出**：已确认的“顶端”玩家列表及其关键优势指标，或经过过滤的最终机会选择（1-3项）。
    *   **常见误区**：仅凭单一指标或主观印象判断；过滤标准模糊或执行不彻底。

3.  **分析优势强化机制与杠杆点**
    *   **输入**：步骤2的输出（顶端玩家列表或最终机会）。
    *   **操作**：
        *   **IF** 分析顶端玩家：探究其优势如何通过正反馈循环自我强化（如规模→研发→更大规模）。
        *   **IF** 作为生态构建者（**路径A**）：基于步骤1的定位，确定自身在生态中的赋能角色（如平台搭建者、标准制定者），并找出1-2个能撬动生态发展的核心杠杆点（关键技术、稀缺资源）。
        *   **IF** 评估合作关系：分析契约边界，识别合作方的核心目标与潜在痛点，找出“契约外可能创造价值的增量项”。
    *   **判断标准**：能够描述正反馈循环、明确核心杠杆点，或区分“履约基线”与“潜在增值点”。
    *   **输出**：优势机制分析、生态角色与杠杆点定义，或“履约基线及潜在增值点”清单。

4.  **制定并执行核心策略**
    *   **输入**：步骤3的分析结论。
    *   **操作**：根据自身角色与目标，选择并执行对应策略：
        *   **策略A（挑战者/新进入者）**：评估是否可通过创新在现有赛道开辟新“顶端”，或进入新兴赛道快速建立优势。
        *   **策略B（顶端玩家/机会锁定者）**：制定巩固和扩大优势的策略，或将主要资源（如20%以上）集中投入已锁定的1-3个高潜力机会。
        *   **策略C（生态构建者 - 路径A）**：利用核心杠杆点，集中资源攻克生态发展的一个关键瓶颈，建立初步优势与声誉。
        *   **策略D（深化合作者）**：评估“潜在增值点”的成本与长期收益，主动执行高性价比的“超越契约行动”，以清晰但非交易性的方式传递善意。
    *   **判断标准**：策略直接回应核心问题（如何获得/利用优势、如何破局、如何建立信任），并具备可执行性。
    *   **输出**：具体的战略行动方案、资源分配计划、破局解决方案或已完成的超越契约行动。

5.  **赋能、监测与迭代**
    *   **输入**：步骤4的执行结果与市场/关系反馈。
    *   **操作**：
        *   **IF** 执行生态构建策略（**策略C**）：将已验证的解决方案产品化/服务化，以赋能其他生态参与者，并从生态繁荣中间接获益。
        *   **IF** 执行投资或竞争策略（**策略A/B**）：持续监测市场信号、趋势及“关键作用力”变化。
            *   **IF** 趋势验证增强 → 考虑追加投入。
            *   **ELSE IF** 出现重大不利变化 → 重新评估，必要时止损或转向。
        *   **IF** 执行合作深化策略（**策略D**）：观察对方反馈。
            *   **IF** 获得更高信任与回馈 → 确认模式有效，持续应用。
            *   **ELSE IF** 被视作理所当然 → 重新评估该方价值，调整投入策略。
        *   **通用行动**：将部分收益或资源再投资于巩固优势或生态建设（如培养人才、支持创新、建立规则）。
    *   **判断标准**：建立了基于反馈的动态调整机制；赋能行为促进了系统繁荣；投资方向与长期目标一致。
    *   **输出**：更新后的策略、调整后的资源分配，或强化后的生态地位与合作关系。

# 输出格式要求

根据初始触发目标，选择以下一种或组合格式输出：

*   **若目标为竞争分析/战略制定**：提供结构化报告，包含：
    1.  **赛道/格局定义**
    2.  **顶端玩家/关键节点分析**
    3.  **优势机制/核心杠杆分析**
    4.  **策略建议与行动计划**
    5.  **监测与迭代要点**

*   **若目标为机会筛选/投资决策**：提供决策摘要，包含：
    1.  **最终选择（1-3项）及筛选逻辑**
    2.  **资源分配计划**
    3.  **后续信号监测要点与动态调仓原则**

*   **若目标为生态构建**：提供战略路径图，包含：
    1.  **生态定位与核心杠杆**
    2.  **破局点与初步优势建立**
    3.  **赋能计划与产品化路径**
    4.  **生态投资承诺**

*   **若目标为深化合作/建立信任**：提供原则框架与行动计划，强调从“识别增量机会”、“评估成本收益”到“主动执行并观察反馈”的闭环过程。

[CRITIC] 合并后的Skill逻辑分支复杂，用户需在步骤1和步骤4根据自身情境（竞争、投资、生态、合作）进行明确选择，否则可能导致路径混淆。建议在实际调用时，由用户或系统首先明确核心触发场景，以简化执行路径。
