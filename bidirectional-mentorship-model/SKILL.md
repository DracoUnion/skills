# bidirectional-mentorship-model

## When to Use

当处于教学、辅导或知识传承关系中，希望突破单向灌输模式，提升双方成长速度时description

## Core Logic

# 执行步骤

1.  **关系基调重设**
    *   **输入**：现有的师徒或教学关系。
    *   **操作**：主动与对方沟通，明确表达“教学相长”的意图。声明关系目标不仅是知识传递，更是共同探索和应对新挑战。强调双方专长领域的差异性价值。
    *   **判断标准**：双方对关系的“双向成长”目标达成口头或书面共识。
    *   **输出**：更新后的关系共识声明。
    *   **常见误区**：导师放不下权威姿态；学员不敢表达不同见解。

2.  **优势领域映射**
    *   **输入**：关系共识，双方的个人背景与技能。
    *   **操作**：各自梳理并明确列出自己的核心优势领域、独特经验或新鲜视角。共同绘制一张“优势地图”，清晰标出谁在哪些方面是“师”，哪些方面是“徒”。
    *   **判断标准**：产生一份可视化的“优势互补图”，双方都认可其准确性。
    *   **输出**：“优势互补图”文档。
    *   **常见误区**：低估学员在新兴领域或跨界视角上的价值；优势描述过于模糊。

3.  **结构化交换议程**
    *   **输入**：“优势互补图”。
    *   **操作**：基于“优势互补图”，共同制定周期性的交流议程。议程应明确两部分：(a) 传统教学部分：由导师主导的知识/技能传授；(b) 反向教学部分：由学员主导的新知分享、案例反馈或跨界视角引入。
    *   **判断标准**：制定出包含明确时间、主题和负责人的近期交流计划。
    *   **输出**：双向教学议程表。
    *   **常见误区**：反向教学部分流于形式或时间被压缩；议题不具体，无法深入。

4.  **创设“安全挑战”环境**
    *   **输入**：双向教学议程。
    *   **操作**：在交流中，导师主动创设允许被质疑和挑战的心理安全环境。学员则需准备有据的、建设性的反馈或替代方案。双方就具体问题（而非人身）进行辩论和推演。
    *   **判断标准**：在交流中能观察到至少一次成功的“角色反转”讨论，即学员就某个议题有效地指导或挑战了导师，并推动了认知深化。
    *   **输出**：关键讨论的记录与 insights。
    *   **常见误区**：讨论演变为个人争执；学员的挑战缺乏依据或建设性。

5.  **共同项目验证**
    *   **输入**：从交流中产生的新想法或解决方案。
    *   **操作**：选择一个小型、具体的共同项目或任务，应用融合双方智慧后形成的新方法去完成。明确分工，共同执行。
    *   **判断标准**：项目得以完成，并且双方都能指出项目中来自对方贡献的关键价值。
    *   **输出**：项目成果报告及双向贡献分析。
    *   **常见误区**：项目过于复杂，难以归因；一方主导，另一方参与度低。

6.  **关系模式复盘与推广**
    *   **输入**：项目成果报告及长期交流记录。
    *   **操作**：定期复盘此双向 mentorship 模式的效果。总结何种议题、何种交流方式最有效。考虑将这种模式固化（如固定“反向汇报”环节），或在更广的团队/组织中推广此理念。
    *   **判断标准**：形成关于如何有效运行双向 mentorship 的几条简明原则或最佳实践。
    *   **输出**：双向 mentorship 模式复盘总结与优化建议。
    *   **常见误区**：不进行复盘，模式无法迭代；将模式机械套用，不注重灵活性。

# 输出格式要求
输出应聚焦于关系动态和协作过程，包含：
1.  **关系定位**：描述双方重新定位后的角色（在A领域为师，在B领域为徒）。
2.  **交换机制**：介绍设立的结构化双向学习议程或例行环节。
3.  **关键突破**：记录一次典型的、成功的“反向教学”或深度碰撞案例。
4.  **共同成果**：展示通过此模式产出的具体项目成果或解决方案。
5.  **模式心得**：总结运行此模式的核心要点和注意事项。
