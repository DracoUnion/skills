# burden-shifting-identification

## When to Use

当分析问题根源或决策后果，怀疑存在责任转嫁或短期行为导致长期成本时description

## Core Logic

# 执行步骤

1.  **界定问题与初步解决方案**
    *   **输入**：一个具体的问题描述（如“公司业绩下滑”、“个人时间不够用”）或一个已提出的解决方案（如“裁员降薪”、“购买时间管理工具”）。
    *   **操作**：清晰定义问题，并记录下表面看来最直接或最省力的解决思路。
    *   **判断标准**：问题和解决方案是否被明确表述。
    *   **输出**：明确的问题陈述和初步解决方案描述。

2.  **追溯责任与成本转移路径**
    *   **输入**：步骤1输出的问题陈述和解决方案描述。
    *   **操作**：分析该解决方案的本质：
        *   **提问**：这个方案是否真正解决了根本原因，还是仅仅将问题的表现（负担）转移给了他人、其他部门、未来或外部环境？
        *   **识别**：找出负担被转移到的具体对象（如员工、客户、未来自己、公共资源）以及被转移的负担类型（如经济成本、心理压力、信任损耗、环境破坏）。
    *   **判断标准**：是否清晰地描绘出“问题产生方 → 解决方案 → 负担承受方”的转移链条。
    *   **输出**：一张负担转移路径图，标明转移的起点、方式和终点。

3.  **评估长期影响与无形成本**
    *   **输入**：步骤2生成的负担转移路径图。
    *   **操作**：
        *   **预测**：评估这种负担转移行为对“承受方”以及最终对“转移方”自身可能产生的长期反噬效果。重点评估对无形资产（如信誉、关系、品牌）的损害。
        *   **计算**：估算因此产生的潜在“无形成本”（如人才流失后招聘成本飙升、客户信任丧失后市场开拓难度倍增）。
    *   **判断标准**：评估是否超越了短期财务损益，涵盖了长期声誉、关系等无形领域的成本。
    *   **输出**：一份风险评估，列出可能引发的长期无形成本及反噬效应。

# 输出格式要求

向用户呈现一份警示性分析，包含：
1.  **分析对象**：针对 [具体问题/方案] 的分析。
2.  **负担转移判定**：明确回答“是否存在负担转嫁”，并简述转移路径。
3.  **被转移的负担**：具体说明哪些成本或问题被转移给了谁。
4.  **潜在无形成本**：列出2-3项最可能发生的长期负面后果，尤其是对无形资产的损害。
5.  **根本解决建议**：提出1-2条指向问题根源、而非转移负担的替代思路。

[CRITIC] 合并后的 Skill 存在逻辑缺陷：步骤3的“评估长期影响与无形成本”仅进行了风险预测和成本估算，但未提供基于此评估的决策或行动建议。整个流程缺少闭环，用户完成评估后，不知道应如何根据评估结果采取行动（例如，是否应否决原方案、采纳替代方案、或制定风险缓解措施）。建议在步骤3后增加一个“决策与行动”步骤，将评估结果转化为具体建议。
