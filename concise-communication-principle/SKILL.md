# concise-communication-principle

## When to Use

当需要准备或传递信息（如写作、演讲、汇报、指导）时，以确保信息有效传达description

## Core Logic

# 执行步骤

1.  **定义沟通目标与受众**
    *   **输入**：沟通任务。
    *   **操作**：明确本次沟通希望达成的具体目标（如告知、说服、指导、建立连接、寻求合作）。明确受众是谁，他们的已知背景、需求和潜在兴趣点。
    *   **判断标准**：能否用一句话清晰描述“希望谁在听后/读后做出何种反应或获得何种知识/建立何种关系”。
    *   **输出**：清晰的沟通目标陈述和受众画像。

2.  **评估自我心态与聚焦点**
    *   **输入**：当前沟通场景的描述、沟通对象、自我对沟通结果的预期。
    *   **操作**：自我审视，判断当前心态属于“积极聚焦”、“消极聚焦”、“积极发散”还是“消极发散”。核心是检查自己是否过度关注结果（消极）或缺乏明确方向（发散），并确立以“积极聚焦”（关注可变过程）为目标状态。
    *   **判断标准**：能够清晰识别出自己当前的主导心态，并意识到“积极聚焦”是目标状态。
    *   **输出**：对自我当前沟通心态的定性判断。

3.  **提炼核心信息点与观察切入点**
    *   **输入**：步骤1的输出，以及你计划传达的所有内容或对沟通场景的初步观察。
    *   **操作**：
        *   **对于信息传递**：围绕沟通目标，筛选出最关键的1-3个信息点。这些信息点必须是支撑目标达成的必要条件。
        *   **对于社交/对话引导**：快速扫描环境与参与者，识别关键人物及其潜在兴趣领域或当前状态，生成初步的“话题切入点”假设列表。
    *   **判断标准**：
        *   **信息点**：每个信息点是否直接、有力地服务于沟通目标。能否删除某个点而不影响目标达成？
        *   **切入点**：能否清晰描述出1-2位关键人物的潜在兴趣领域或当前状态。
    *   **输出**：按重要性排序的核心信息点列表，或一个/多个初步的“话题切入点”假设列表。

4.  **构建表达结构或发起对话**
    *   **输入**：步骤3输出的核心信息点列表或话题切入点假设。
    *   **操作**：
        *   **对于信息传递**：为信息点设计一个逻辑递进的结构（如问题-方案、结论-理由、步骤1-2-3）。为每个信息点准备最精炼的论述或例证。
        *   **对于社交/对话引导**：选择一个与对方（尤其是关键人物）明显相关且非隐私的话题切入点，以提问或观察评论的方式开启对话。例如：“听说您最近在忙[某项目]，进展如何？”或“您对刚才提到的[某观点]怎么看？”
    *   **判断标准**：
        *   **结构**：是否自然流畅，易于受众跟随。每个部分的论述是否紧扣其信息点，无枝蔓。
        *   **对话发起**：问题是否以对方为中心，是否开放且易于回答，是否能引发对方的表达欲。
    *   **输出**：信息传达的简要提纲，或对方的初始反应（积极、中性、拒绝）。

5.  **建立合作前提与调整聚光灯**
    *   **输入**：步骤2的自我心态判断、步骤4的输出（提纲或对方初始反应）、沟通目标。
    *   **操作**：
        *   **A. 应用合作原则（建立“合力为正”基础）**：在沟通中主动应用并展示以下四个原则：
            *   **透明**：清晰表达自己的想法、需求和顾虑。
            *   **坦荡**：保持行为与原则的一致性。
            *   **鼓励**：对合作过程中的尝试与努力给予正向反馈。
            *   **补偿**：当对方表现超出预期时，主动给予认可。
        *   **B. 评估反应并引导对话**：
            *   **IF 对方表现出兴趣（如详细回答、反问、情绪积极）**：将聚光灯持续聚焦于对方。深入追问细节、请求举例、或表达共鸣，引导对方多讲述。
            *   **ELSE IF 对方反应中性或简短**：轻微调整话题角度，尝试另一个相关切入点，或引入第三方事例延续话题。
            *   **ELSE IF 对方明显不感兴趣或拒绝**：礼貌中止当前话题，转向环境或其他中性话题。
    *   **判断标准**：
        *   **合作氛围**：沟通氛围是否趋向开放、互信，能就事论事。
        *   **对话质量**：对话是否能够以对方为主角持续进行；对方是否表现出更多参与感。
    *   **输出**：观察到的对方初步反应及合作氛围评估，或确认的“有效对话路径”/“无效信号”。

6.  **判断合作可行性并采取行动**
    *   **输入**：步骤5的输出，观察对方在沟通中的长期行为模式（是否透明、坦荡、有改进意愿）。
    *   **操作**：
        *   **IF** 对方行为模式基本符合合作原则，或展现出改进意愿 -> **继续推进**。对于信息传递，进入步骤7；对于对话引导，在对方充分表达后，寻找共鸣点并克制分享自身相关信息，完成“投射-共鸣-回馈”循环。
        *   **ELSE** (对方持续表现出不透明、反复无常、消耗资源且无改进意愿) -> **执行拒绝与边界设定**：
            *   明确告知对方，由于其当前行为模式，现有合作或关系模式无法继续。
            *   声明拒绝是针对“当前不可持续的行为模式”，只要对方愿意改变行为、重启基于四项原则的沟通，合作可以重新开始。
    *   **判断标准**：能够基于观察到的行为客观判断合作可持续性；拒绝信息是否清晰、坚定、对事不对人。
    *   **输出**：继续合作或启动拒绝流程的决策。若继续，输出深化后的沟通内容或成功建立的初步连接。

7.  **冗余检查与预演调整（针对信息传递）**
    *   **输入**：步骤6中继续推进后产生的沟通内容或终版提纲。
    *   **操作**：逐部分检查并删除以下内容：与核心信息点无关的细节、重复表述、复杂术语（除非必要）、过多背景铺垫、用于显示学识而非服务目标的额外内容。随后模拟向目标受众进行传达，检查时长、节奏和清晰度。
    *   **判断标准**：
        *   **精简度**：删除后，核心信息是否更突出、结构是否更紧凑？
        *   **预演效果**：
            *   **IF** 预演顺利，预计能有效达成目标 -> 流程结束，准备正式传达。
            *   **ELSE IF** 发现仍有冗余或模糊处 -> 返回本步骤进行进一步删减或澄清。
            *   **ELSE** (发现结构或核心信息点有问题) -> 返回步骤3或4进行调整。
    *   **输出**：最终确定的沟通方案。

# 输出格式要求

根据沟通的主要类型，提供相应的方案摘要：

**A. 信息传递型沟通：**
1.  **沟通目标**：[一句话描述]
2.  **目标受众**：[简要描述]
3.  **核心信息点**：
    *   1. [点一]
    *   2. [点二]
    *   3. [点三] (如有)
4.  **表达结构**：[如“总分总”、“问题-分析-方案”]
5.  **精简提示**：[重点提醒需避免的冗余类型]

**B. 关系建立/引导型沟通：**
1.  **对话评估**：本次沟通是否成功启动了以对方为中心的对话循环。
2.  **关键发现**：获取了关于对方的哪些关键信息、观点或需求。
3.  **关系状态**：初步连接是否建立，以及下一步互动的潜在方向建议。

**C. 高压/合作型沟通：**
1.  **核心理念**：积极聚焦，建立基于“透明、坦荡、鼓励、补偿”原则的合作。
2.  **动作建议**：具体如何体现四项原则。
3.  **边界设定（如需要）**：提供坚定而清晰的拒绝话术框架，并解释拒绝是为了终止消耗性模式，为未来健康的重新连接保留可能。

[CRITIC] 合并后的步骤6中，“继续推进”分支同时指向信息传递的后续步骤（步骤7）和对话引导的“完成循环”动作，逻辑上存在并行路径，在单一线性步骤描述中可能造成混淆。建议在步骤6的“继续推进”分支内明确两个子分支。
