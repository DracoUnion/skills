# contextual-insight-extrapolation

## When to Use

当获得特定情境下的“毛金信息”（一手事实）后，需要将其转化为更广泛的指导原则或应用于新情境时description

## Core Logic

# 执行步骤

1.  **解构原始情境的成功要素**
    *   **输入**：来自“raw-information-extraction”模型的输出报告（包含核心洞察与边界说明）。
    *   **操作**：分析报告中的“核心洞察”和“原始事实还原”。将成功结果分解为几个关键构成要素（如：特定用户行为、未被满足的需求、独特的时间窗口、关键人物的决策等）。
    *   **判断标准**：是否将具体的成功案例抽象成了3-5个可分离的关键变量或条件。
    *   **输出**：一份成功要素清单。

2.  **对比目标情境的差异点**
    *   **输入**：成功要素清单；当前需要决策或分析的新情境描述。
    *   **操作**：将成功要素清单中的每一项，与新情境的对应条件进行逐一对比。识别哪些要素在新情境中**存在**、**类似但不同**、**缺失**或**相反**。
    *   **判断标准**：是否清晰识别出至少一项关键差异点。
    *   **输出**：一份要素对比分析表。

3.  **进行适应性变通与重构**
    *   **输入**：要素对比分析表。
    *   **操作**：
        *   IF 某个关键成功要素在新情境中**缺失**或**相反**，THEN 评估该要素是否为核心必要条件。若是，则提示“直接复制风险高”，需寻找替代方案或重新评估机会。
        *   IF 要素**存在**或**类似**，THEN 思考如何在新情境中激活或适配该要素。设计具体的、符合新情境的替代动作或策略。
    *   **判断标准**：是否为存在差异的要素提出了具体的、情境化的适配方案或风险警示。
    *   **输出**：一套针对新情境的、经过变通后的行动假设或策略选项。

4.  **形成可验证的假设**
    *   **输入**：经过变通后的行动假设或策略选项。
    *   **操作**：将策略转化为一个或多个可被验证的假设（例如：“如果我们采取X行动，将会产生Y结果，因为Z要素与原始情境类似”）。同时，明确验证该假设所需的最小化测试或关键观测指标。
    *   **判断标准**：假设是否清晰、可被后续行动验证。
    *   **输出**：一份包含变通策略、背后逻辑及验证方法的可行性假设报告。

# 输出格式要求

以对比分析与建议方案形式呈现：
1.  **原始洞察来源**：[简述引用的“毛金信息”案例]
2.  **原始成功要素**：[列出解构出的关键要素]
3.  **新情境对比**：
    *   要素A：新情境中 [存在/类似/缺失/相反]，具体表现为...
    *   要素B：...
4.  **变通策略与假设**：
    *   针对要素A的差异，建议采取 [具体行动]，预期产生 [结果]，因为 [逻辑]。
    *   验证方法：通过 [具体测试或指标] 来验证此假设。
5.  **总体风险评估**：[基于关键要素的匹配度，评估直接复制结果的风险等级（高/中/低）及主要依据]
