# dual-perspective-decision-filter

## When to Use

当处于极端情境（极度顺利或极度困难）时，用于强制进行反向思考，避免决策盲区description

## Core Logic

# 执行步骤

1.  **识别当前情绪极点**
    *   **输入**：对自身或组织当前处境的整体感知。
    *   **操作**：判断当前主要情绪和情境是偏向“至暗时刻”（如绝望、失败、危机）还是“至明时刻”（如狂喜、成功、顺境）。
    *   **判断标准**：能够明确将当前状态归类到其中一个极点。
    *   **输出**：当前主导的极点标签（“至暗”或“至明”）。

2.  **执行强制性反向追问**
    *   **输入**：步骤1输出的极点标签。
    *   **操作**：
        *   **IF** 当前为“至暗时刻”：追问“前方可能存在的转机、机遇或长期价值是什么？”（向前看）
        *   **IF** 当前为“至明时刻”：追问“为此成功，已经或即将付出的隐藏代价是什么？”（反向想）
    *   **判断标准**：提出了一个与当前主导情绪相反方向的、具体的探索性问题。
    *   **输出**：一个反向探索性问题。

3.  **系统搜集反向证据**
    *   **输入**：步骤2输出的反向探索性问题。
    *   **操作**：基于该问题，从历史案例、数据、他人经验、逻辑推演等渠道，搜集支持或否定该问题的证据与可能性。
    *   **判断标准**：搜集到至少两条相关的、有依据的信息或推论。
    *   **输出**：一份关于反向可能性的证据清单。

4.  **整合视角并调整认知**
    *   **输入**：步骤1的极点标签和步骤3的证据清单。
    *   **操作**：将反向证据与当前主导认知进行整合，形成一个新的、更平衡的综合性判断。
    *   **判断标准**：新的判断不再单一偏向初始极点，而是包含了反向视角的元素。
    *   **输出**：一个经过视角平衡后的修正认知或判断。

# 输出格式要求
以简短的决策备忘录形式呈现，包含：
1.  **初始判断**：识别的极点（至暗/至明）及简要描述。
2.  **强制反向问题**：提出的具体追问。
3.  **反向证据摘要**：列举发现的关键反向可能性或代价。
4.  **平衡后观点**：整合正反视角后的新结论或待办事项。
