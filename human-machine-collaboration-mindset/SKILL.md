# human-machine-collaboration-mindset

## When to Use

当评估个人职业发展、技能规划或思考在自动化/人工智能趋势下的个人定位时description

## Core Logic

# 执行步骤

1.  **【环境与结构分析】识别宏观趋势与竞争本质**
    *   **输入**：对目标领域/社会/市场的初步认知，以及长期数据（如收入、市场份额分布）。
    *   **操作**：
        a. **分析资源分配结构**：收集数据，绘制关键指标（收入、财富、市场份额）的分布图。判断形态为“橄榄型”（正态）、“M型”（去中间化）或其他。若为“M型”，验证其是否在子系统（分蘖）中复现，并评估资源重新分配的频率（高频/低频）。
        b. **识别竞争模式**：诊断系统内主要参与者的互动是“争”（内部瓜分有限资源）还是“竞”（联合向外拓展）。预测当前模式对系统健康度（走向平庸或繁荣）的长期影响。
        c. **探查真实价值地带**：超越表层竞争（如热门技能、产品），通过深度参与和连接内行，探查决定利润、影响力或成败的真实关键地带（如政策、生态位、供应链）。评估该地带的价值性、决定性、隐蔽性与持续性。
    *   **判断标准**：形成对所处环境结构性特征（M型/橄榄型、争/竞）、动态机制及真实高价值区域的清晰认知。
    *   **输出**：一份包含分布形态诊断、竞争模式分析及关键竞争地带定义的环境分析报告。

2.  **【自我审计与定位】评估个人资产、能力与角色**
    *   **输入**：个人当前的核心技能、工作内容、资源状况及步骤1的环境分析结果。
    *   **操作**：
        a. **技能与资产审计**：
            *   **可替代性分析**：区分“高重复性/可模式化”的左脑任务（面临自动化风险）和“需要审美、叙事、共情、复杂决策”的右脑任务（人力优势）。
            *   **资产/负债判定**：罗列有形与无形资源（技能、品牌、关系、知识），判断每项是能持续创造价值的“资产”，还是消耗维护成本、引发未来问题的“负债”（无形成本）。
        b. **定义个人终极角色**：
            *   避免简单模仿成功者的“手段”（专业、头衔），追溯其解决的“原始问题”或抓住的“根本机遇”。
            *   基于环境分析（关键竞争地带、竞争模式）和自身“资产”，定义自己在新的协作网络中的核心功能角色（如“信息整合者”、“生态构建者”），而非静态身份标签。
        c. **评估均值回归压力**：审视个人关键表现指标（如业绩增长率、能力评分）的历史数据，判断当前是否处于异常高/低表现期，预判下一阶段可能面临的向上或向下回归压力。
    *   **判断标准**：清晰列出“需防御/自动化风险高”、“需增强/人机协作点”、“纯人力优势/核心竞争力”三类技能；明确主要无形资产与无形成本；提出功能化、非模仿性的个人角色陈述。
    *   **输出**：个人技能分析矩阵、资产/负债清单、角色定位陈述及增长趋势预判。

3.  **【战略路径设计】选择发展模式与协作框架**
    *   **输入**：步骤2的自我定位、步骤1的环境分析，以及个人/事业长期愿景。
    *   **操作**：
        a. **选择核心发展路径**：
            *   分析事业愿景所涉关键参与方及其互动网络。若系统简单、线性（参与方≤2，关系固定），则采用 **MVP迭代路径**，从最小可行产品开始快速优化。
            *   若系统复杂、动态、追求生态或平台效应（参与方≥3，关系多向影响），则必须采用 **顶层设计路径**，先构思系统架构、规则与核心闭环，再分阶段实施。
        b. **设计人机协作模式**：针对“需增强”的技能点，具体设计“人”与“机器/工具”的分工流程。明确机器负责处理（数据、重复执行），人负责处理（目标设定、解读、创意、决策、情感互动），追求“1+1>2”。
        c. **制定“竞”模式策略**：若环境处于“争”模式而希望转向“竞”，设计策略：将竞争目标转向外部市场；建立动态平衡机制让强弱流动；设计联盟利益共享机制；构建分层成长与流通体系。
    *   **判断标准**：发展路径选择与系统复杂性匹配；人机分工明确且优势互补；促“竞”策略具体可操作。
    *   **输出**：明确的发展路径选择（MVP/顶层设计）、具体的人机协作蓝图、以及（如需要）促“竞”策略方案。

4.  **【能力进化与学习】规划系统性成长**
    *   **输入**：步骤3设计的协作模式与战略路径，以及个人角色定位。
    *   **操作**：
        a. **能力进化规划**：基于“能力分包”趋势（如记忆分包给云盘，诊断分包给AI），识别个人可能退化的能力（如死记硬背）和必须进化的能力（如信息甄别、资源整合、AI提示工程、社群运营）。
        b. **学习投资评估**：对待学习的新技能/知识（S），进行三层评估：
            *   **关联性筛查**：S是否直接服务于个人核心属性或角色？
            *   **系统性评估**：S是孤立技能点，还是能与现有知识体系形成协同的“系统组件”？
            *   **ROI估算**：对比学习投入与对核心能力及无形资产（荣誉、成果）的预期提升。
        c. **制定整合学习方案**：将通过评估的技能学习，与现有项目或“组合验证项目”结合，明确实践场景、目标及与已有能力的组合应用方式。
    *   **判断标准**：进化方向聚焦于管理外包和创造新价值；学习决策基于系统性价值和ROI，而非热点；学习计划有明确的实践挂钩点。
    *   **输出**：能力退化预警与进化方向列表、具体技能的学习建议（立即/延后/不学）及整合学习方案。

5.  **【执行与决策】应用高效决策模型**
    *   **输入**：具体任务、合作选择或投资评估场景。
    *   **操作**：
        a. **效率与直觉决策**：当面临“亲自体验”与“授权专家”的选择时，明确任务核心目标。若授权专家的总成本更低且能更好达成目标，则选**效率中心**路径；若过程体验是核心目标或无可信专家，则选**直觉中心**路径。
        b. **天使投资式评估**：当需快速评估支持某个体（投资、合作、选拔）时，基于其陈述，评估五个维度信号：
            *   **安全感**：重视并维系长期稳定关系。
            *   **资格感**：对自身价值与潜力有健康自信。
            *   **三观一致**：行为逻辑追求多方共赢的正循环。
            *   **目标感**：长期专注一个方向，且方法灵活迭代。
            *   **谦虚诚恳**：常怀感恩，坦然承认未知。
        c. **趋势直觉感知**：快速评估新兴事物价值或群体情绪时，先记录快速直觉判断，再寻找1-2个证据支持或挑战该直觉，最后做出轻量级行动决策（尝试、观察或搁置）。
    *   **判断标准**：决策模型与场景匹配；评估基于多维度信号而非单一因素；行动决策明确。
    *   **输出**：明确的决策选择（效率/直觉）、个体支持建议（高优先级/可考虑/高风险）及理由、或针对新兴事物的行动指令。

6.  **【心智与叙事】调整心态并构建影响力**
    *   **输入**：对技术变革的整体态度、个人过往经历、及当前发展阶段。
    *   **操作**：
        a. **心智模式自检与切换**：
            *   自检：面对变化，默认反应是积极兼容、置身事外还是消极对抗？明确自身心智模式。
            *   **IF** 发现固有性格/习惯成为新角色障碍，**THEN** 执行“角色切换”：将旧有特质视为已完成使命的“助推器”准备分离；定义新角色所需的功能性行为；通过仪式感进入“情境自我”（me）；事后评估效能并回收经验，保持灵活性。
        b. **构建个人故事框架**：使用“起承转合”结构规划个人叙事：
            *   **起**：确立真实、具体的起点事件。
            *   **承**：描述由此引发的情感与初步状态，保持连贯。
            *   **转**：构建有力的核心转折点，体现冲突与改变。
            *   **合**：收束故事，将经历升华为具有普遍意义的成长与价值。
        c. **维护个人成长报表**：定期（如每季度）更新：
            *   **行为负债表**：盘点“知识/能力债务”，制定偿债计划。
            *   **价值流量表**：分析资源、人脉的流入流出与圈层变化。
            *   **综合利润表**：计算无形资产增值减去投入成本后的“盈余”，并估算成长增长率与个人“市盈率”（市场价值预期）。
    *   **判断标准**：能主动调整心态适应角色；个人故事真实、有感染力；成长报表能持续反映趋势并指导调整。
    *   **输出**：心智模式自检结果、个人故事框架大纲、周期性的个人成长三张报表及核心指标。

# 输出格式要求

根据触发场景的不同侧重点，输出相应的结构化报告。通用报告应包含：

1.  **环境与结构分析摘要**：关键发现（如M型结构、关键竞争地带）。
2.  **个人审计与定位结果**：技能分析矩阵、核心角色陈述、主要资产与负债。
3.  **战略与协作设计**：选择的发展路径及理由、人机协作蓝图要点。
4.  **能力进化与学习计划**：关键进化方向、下一阶段学习重点及整合方案。
5.  **核心决策建议**：基于当前分析，给出的1-3项最高优先级行动或决策建议。
6.  **心智与成长指标**：心智模式状态、个人故事核心线索、近期成长增长率趋势。

[CRITIC]
1.  **逻辑闭环缺陷**：步骤6的“心智与叙事”部分，其输出（如个人故事、成长报表）如何直接反馈并影响步骤1的“环境分析”或步骤2的“自我审计”，形成闭环迭代机制，描述不够清晰。模型整体是线性递进式，缺乏明确的“复盘-调整”循环触发点。
2.  **分支处理遗漏**：在步骤2c“评估均值回归压力”后，预判了“向上/向下回归”，但步骤4“能力进化与学习”中并未明确针对不同回归压力（尤其是向下回归）制定差异化的“防御与巩固”或“布局与投资”策略，存在逻辑断点。
3.  **错误处理缺失**：模型未涉及关键环节失败或误判时的回退或纠错机制。例如，步骤3中选择“顶层设计”路径后，若发现早期设计存在根本缺陷，应如何低成本调整或转向？步骤5中“天使投资式评估”出现误判导致损失后，如何更新评估框架？
