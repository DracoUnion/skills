# idea-integration-before-creation

## When to Use

当获得一个灵感或初步想法时，在深入发展或执行前，先进行外部信息整合。description

## Core Logic

# 执行步骤

1.  **灵感暂存与目标定义**
    *   **输入**：脑海中闪现的灵感或初步想法；或一个待解决的创造性问题/已发生的创造性成果实例。
    *   **操作**：
        *   **IF** 输入是灵感：立即记录下该灵感（关键词或简短描述），但**不**立即进行深入发散。此记录即为“创造标的”。
        *   **IF** 输入是问题或案例：用一句话清晰描述要达成的创造性目标，或选定要分析的创造性案例。此描述即为“创造标的”。
    *   **判断标准**：灵感被准确记录，未丢失核心信息；或目标/案例描述具体，非泛泛而谈。
    *   **输出**：明确的“创造标的”。

2.  **外部信息检索**
    *   **输入**：步骤1中记录的“创造标的”（灵感关键词或问题/案例描述）。
    *   **操作**：以此为核心关键词，进行系统性信息检索。目标包括：寻找是否有他人提出过类似想法、更成熟的案例、更精准的概念表述、相关的数据或理论支持。
    *   **判断标准**：收集到至少3-5条与“创造标的”相关的外部信息（文章、案例、观点等）。
    *   **输出**：相关的外部信息集合。

3.  **创造力模式匹配**
    *   **输入**：“创造标的”。
    *   **操作**：根据“创造标的”的特征或发生情境，将其与以下三种创造模式进行匹配：
        *   **模式A（灵感迸发）**：其核心是“颅内神经元连接”。特征表现为非连续、顿悟式，通常发生在放松、非专注状态（如散步、沐浴），连接的是大脑内部已有的、看似无关的信息模块。
        *   **模式B（厚积薄发）**：其核心是“人际关系连接”。特征表现为基于长期、有意识的积累和社交网络互动，连接的是不同人的知识、经验和视角模块。
        *   **模式C（即兴而发）**：其核心是“场景连接”。特征表现为对当下环境、随机事件或即时可用资源的快速反应和重组，连接的是特定场景中的元素模块。
    *   **判断标准**：能够将“创造标的”合理归类到至少一种主要模式，并解释其连接的本质。
    *   **输出**：对“创造标的”所属“创造模式”的判定及理由。

4.  **信息比对、整合与策略制定**
    *   **输入**：“创造标的” + 步骤2收集的外部信息 + 步骤3判定的“创造模式”。
    *   **操作**：
        *   **a) 信息比对与整合**：将外部信息与“创造标的”进行比对。识别：i) 哪些部分已被他人更好解决或表达（可直接采纳或借鉴）；ii) 哪些部分是“创造标的”的独特之处或潜在优势；iii) 不同信息之间是否存在可连接、组合的空间。
        *   **b) 应用或复盘模式策略**：
            *   **IF** 目标是激发新创造（前瞻性应用）：
                *   根据判定的模式，采取对应的策略：
                    *   若追求**模式A**：安排不聚焦的放松时间，接触多样化信息，等待并记录灵感。
                    *   若追求**模式B**：有目的地构建跨领域人际网络，进行深度交流与知识整合。
                    *   若追求**模式C**：主动进入新场景，设定约束条件，练习对即时元素的快速组合与迭代。
            *   **IF** 目标是分析已有案例（后验性复盘）：
                *   根据判定的模式，逆向分析其成功的关键连接点是什么（哪些内部概念、哪些人的贡献、哪些场景元素被连接了起来）。
    *   **判断标准**：
        *   （整合）能够清晰指出“创造标的”的“已知/可借鉴部分”和“未知/待创新部分”。
        *   （应用时）制定了符合所选模式的具体行动项；（复盘时）清晰地指出了促成创造的核心连接点。
    *   **输出**：整合分析结论 + 一套针对性的“创造策略”或一份“创造过程复盘分析”。

5.  **迭代、重构与决策**
    *   **输入**：经过比对整合后的想法组件、模式策略。
    *   **操作**：用更优的外部案例或表述替换“创造标的”中相对薄弱的部分。在此基础上，聚焦并深化那些独特的、有优势的部分，结合选定的创造模式策略，形成迭代后的新想法框架或行动计划。
    *   **判断标准**：产出的想法框架比初始灵感更扎实、更清晰，且包含了外部验证、模式化策略优化的元素。
    *   **IF** 整合后想法价值显著提升，且模式策略可行：
        *   **输出**：进入下一步的详细规划或“自主创新验证”流程。
    *   **ELSE** 整合后发现想法价值不大、已被充分实现，或模式匹配度低、策略难以执行：
        *   **输出**：放弃或大幅调整初始方向，避免在低价值或不可行路径上浪费精力。

# 输出格式要求

以“创意整合与模式分析报告”格式呈现：
1.  **原始标的**：记录步骤1定义的“创造标的”。
2.  **外部参考**：列出步骤2收集的关键信息源及核心观点。
3.  **模式判定**：陈述步骤3的判定结果（所属创造模式及理由）。
4.  **整合分析与策略**：
    *   **可借鉴部分**：来自步骤4a。
    *   **待深化/独特部分**：来自步骤4a。
    *   **创造策略/复盘要点**：来自步骤4b（根据应用或复盘目标选择其一）。
5.  **迭代后框架与决策**：呈现步骤5产出的新想法概要或行动计划，并明确最终的推进或调整决策。

[CRITIC]
1.  **逻辑闭环缺陷**：步骤5的决策分支（IF/ELSE）中，“价值显著提升”和“模式策略可行”的判断标准未在步骤中明确定义，依赖于执行者的主观判断，可能导致决策不一致。
2.  **潜在矛盾未处理**：步骤1将“灵感”和“问题/案例”统一处理为“创造标的”，但后续步骤（如外部信息检索）对“已发生的创造性案例”进行检索的目标和有效性可能存在疑问，未做区分说明。
3.  **步骤复杂度**：合并后的Skill步骤较多（5步），且每一步内含判断与操作，对用户的认知负荷要求较高，可能影响执行流畅度。
