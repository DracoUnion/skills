# identify-singularity-trend

## When to Use

当观察技术、社会或组织发展历程，需要判断哪些节点是革命性突破（奇点）而非渐进式改进时description

## Core Logic

# 执行步骤

1.  **梳理发展序列**：列出所观察领域从早期到当前的关键发展阶段或里程碑事件。
    *   **判断标准**：序列能大致反映该领域的时间演进脉络。
    *   **常见误区**：遗漏关键过渡阶段或混淆不同层级的事件。

2.  **分析变化模式**：针对序列中的每个阶段或事件，分析其相对于前一阶段带来的变化性质。主要判断是“性状/结构改变”还是“数量/效率的线性提升”。
    *   **判断标准**：能够清晰区分某次变化是引入了新范式、新结构（如从单细胞到多细胞），还是对原有范式的优化（如存储容量从兆到千兆）。
    *   **常见误区**：将重要的线性提升误判为结构性变革。

3.  **识别聚合前置条件**：对于被初步判断为“性状/结构改变”的节点，检查其发生前是否存在不同元素、思想或技术进行“聚合”的迹象。例如，不同学科的交叉、多种技术的整合、多元思想的交汇。
    *   **判断标准**：能够识别出导致突变发生的、具体的、异质元素的聚合现象。
    *   **常见误区**：将时间上的先后关系误认为因果聚合关系。

4.  **确认非线性效应**：验证该节点带来的后续影响是否是“非线性”的。即，其是否开启了全新的发展曲线，其影响力或发展速度是否远超之前的线性累积趋势。
    *   **判断标准**：有证据表明该节点之后，领域的发展轨迹、规模或复杂度发生了阶跃式变化。
    *   **常见误区**：将短期热点误判为长期的非线性奇点。

5.  **标注奇点**：综合步骤2-4的分析，将同时满足“由聚合引发”、“导致性状/结构改变”、“产生非线性效应”的节点标注为该领域的“奇点”。
    *   **判断标准**：标注结果符合“因聚合而突变，效率和结构的改变是非线性的”核心定义。
    *   **常见误区**：标注过多或过少的节点，失去识别“划时代里程碑”的意义。

# 输出格式要求
当用户需要分析某个领域的发展关键点或寻找创新机会时，提供此模型。输出应强调“聚合”、“结构性改变”和“非线性效应”三个判断维度的综合运用，帮助用户区分真正的革命性突破与重要的渐进式改进。
