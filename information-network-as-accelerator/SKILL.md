# information-network-as-accelerator

## When to Use

当处于快速变化的环境，需要持续获取前沿知识、趋势或最佳实践以保持竞争优势或实现创新时，调用此模型来构建和利用高效信息网络。description

## Core Logic

# 执行步骤

1.  **明确核心信息需求与目标**
    *   **输入**：当前面临的关键决策、创新方向、知识短板或待解决的问题。
    *   **操作**：将模糊的需求转化为具体、可检索、可评估的问题或目标（例如：“获取本季度某行业的一手销售数据”、“降低XX成本的最新工艺”）。
    *   **判断标准**：需求是否具体到可以用于指导信息源搜寻和纯度评估。
    *   **输出**：“核心信息需求与目标清单”。
    *   **常见误区**：需求过于宽泛，导致后续评估标准模糊。

2.  **地图化与预筛高价值信息源**
    *   **输入**：“核心信息需求与目标清单”。
    *   **操作**：针对每个需求，寻找并标注潜在的高价值信息源，并依据客观特征进行初步纯度分级：
        *   **高纯度候选源（纯度 > 60%）**：产生原始知识或数据的个人、机构、前沿社区、经过严格同行评议的学术期刊、专利库、官方统计数据。
        *   **中纯度候选源（纯度 30%-60%）**：有效筛选、整合并传播一级源信息的平台、权威媒体或机构的深度分析文章、知名专家的公开深度论述、基于原始数据的第三方研究报告。
        *   **低纯度候选源（纯度 < 30%）**：网络小说、标题党文章、情绪化叙述为主的内容、未经核实的二手信息、大部分娱乐化信息流内容。
    *   **判断标准**：为每个核心信息需求找到至少一个高纯度或中纯度候选源。
    *   **输出**：一份附有初步纯度评级的“信息源地图”。
    *   **常见误区**：被信息来源的表面权威性或知名度迷惑，忽略其内容的生产过程和实质。

3.  **评估单条信息纯度（深度审查）**
    *   **输入**：从“信息源地图”中选取的待评估具体信息内容（文章、报告、数据、观点等）。
    *   **操作**：执行深度纯度审查，按顺序检查：
        *   **3.1 形式与动机筛查**：快速浏览标题、结构、呈现形式。识别是否存在强烈的个人情绪词汇、极端化论断、煽动性语言、明显的立场预设或动机不纯的渲染。**IF** 存在上述特征 **THEN** 质疑其客观性，标记“杂质含量高”。**ELSE** 进入下一步。
        *   **3.2 核心内容提炼**：尝试在1-2分钟内提炼信息的核心主张及1-3个关键支撑论据。**IF** 提炼成功且论据与主张关联紧密 **THEN** 进入步骤3.3。**ELSE IF** 核心主张模糊、论据薄弱或结构混乱导致难以提炼 **THEN** 执行“冗余内容剥离”（标记重复表述、无关背景、情绪化表达等），再看剩余部分能否提炼。若仍不能，标记为“低纯度信息”。
        *   **3.3 价值密度与验证检查**：评估提炼出的核心内容：
            *   **证据链**：结论是否有数据、实验或逻辑严密的推理支持？
            *   **时效性与一手性**：信息是否最新？是否接近原始数据或事件？
            *   **利益声明与客观性**：发布者是否有未声明的明显偏见？
            *   **可证伪性**：观点是否清晰且可被事实检验？
            *   **对需求的适用性与新颖性**：是否直接针对步骤1的需求，并提供新的视角或解决方案？
    *   **判断标准**：通过上述检查后，给出最终纯度评级（高/中/低）及理由。
    *   **输出**：对单条信息的“纯度评估报告”（包含形式纯度、核心提炼、价值纯度评级及理由）。

4.  **决策、整合与网络构建**
    *   **输入**：步骤3的“纯度评估报告”及步骤2的“信息源地图”。
    *   **操作**：
        *   **4.1 信息采纳决策**：
            *   **IF** 信息纯度评级为“高”，且直接满足步骤1的需求 **THEN** 将其作为核心证据或关键输入。
            *   **ELSE IF** 信息纯度评级为“中”，或高纯度信息不完全匹配需求 **THEN** 将其作为辅助参考，需与其他来源交叉验证。
            *   **ELSE** (信息纯度评级为“低”) **THEN** 原则上舍弃，或仅作为了解大众观点的渠道。
        *   **4.2 建立信息流入与筛选机制**：基于“信息源地图”和纯度评估经验，设计定期获取信息的流程（如订阅高纯度源、设置关键词提醒），并建立初步筛选标准（如优先看被高纯度源引用的、解决具体问题的内容）。
        *   **4.3 构建加工与模式识别能力**：对采纳的高纯度信息进行深度加工（交叉验证、追溯源头），主动寻找信息之间的关联、矛盾或重复出现的模式、趋势。
        *   **4.4 应用洞察与闭环反馈**：将提炼的洞察应用于实际决策或创新中。**IF** 应用成功 → 强化相关信息源的关注。**ELSE IF** 未能奏效或环境变化 → 返回步骤1，重新校准信息需求。
    *   **判断标准**：决策与信息网络构建是否基于高纯度信息，且网络能根据实践反馈持续优化。
    *   **输出**：信息采纳结论、一个运行的“个性化信息网络建设蓝图”及后续优化方向。

# 输出格式要求

根据触发场景的侧重点，输出以下两种格式之一：

**A. 当侧重单条信息评估时，输出“信息纯度评估报告”：**
1.  **信息源**：[标题或来源]
2.  **形式纯度**：[高/中/低] - [基于步骤3.1的说明]
3.  **核心提炼**：[1-2句话概括的核心主张与关键论据]
4.  **价值纯度**：[高/中/低] - [基于步骤3.3的说明，如证据链、时效性、对需求的适用性等]
5.  **处理建议**：[直接采纳 / 剥离情绪后参考并交叉验证 / 忽略]

**B. 当侧重构建信息网络时，输出“个性化信息网络建设蓝图”：**
1.  **核心需求**：基于用户场景提炼的1-3个核心信息需求。
2.  **关键信息源地图**：为每个需求推荐的、附有纯度评级的高价值信息源（高/中纯度候选源各1-2个）。
3.  **流入与筛选机制**：建议的日常信息获取与初步处理流程。
4.  **预期产出与原则**：说明该网络有望帮助发现的洞察类型，并重申基于高纯度信息的核心优势（认知效率、决策质量、情绪稳定、时间效益）及衍生的处理原则。

[CRITIC] 步骤3.2（核心内容提炼）的“ELSE IF”分支中，若信息因结构混乱难以提炼，流程指示执行“冗余内容剥离”后再次尝试。然而，若剥离后仍无法提炼核心，则标记为“低纯度信息”并结束。此处逻辑完整，但未明确说明在“冗余内容剥离”子步骤中若发现信息本质空洞，是否可直接判定为低纯度并提前终止流程。建议在操作描述中增加：“若在剥离过程中发现信息本质空洞、缺乏实质性内容，则可直接判定为‘低纯度信息’，流程终止。”
