# information-source-classification-framework

## When to Use

当需要系统性地建立或整理个人/组织的高质量信息来源库时description

## Core Logic

# 执行步骤

1.  **定义信息质量评估标准**
    *   **输入**：对信息价值的通用理解、你的学习/成长目标、目标领域知识。
    *   **操作**：建立并书面化一个适用于自身领域的信息质量分级标准。标准应包含两个维度：
        *   **维度一：信息纯度层级**
            *   **层级A (高纯度 >60%)**：原始数据源、经同行评议的学术期刊、领域顶尖专家的一手经验与思考。
            *   **层级B (中纯度 30%-60%)**：权威机构/媒体的深度行业报告、知名专家的系统性公开输出、优质书籍。
            *   **层级C (低纯度 <30%)**：大众娱乐内容、未经严格验证的二手观点、以吸引流量为主要目的的内容。
        *   **维度二：个体认知密度特征**（用于评估信息生产者）
            *   特征应为客观、可观察、可验证的行为或成果，例如：
                *   在特定领域有多次成功决策/操盘记录。
                *   其公开发表的观点/文章信息浓度高，常能提炼出可操作的模型。
                *   其行为模式展现出“先舍后取”的明确取舍逻辑。
    *   **判断标准**：标准是否清晰、可操作，能对不同类型信息及信息生产者进行相对一致的归类与识别。
    *   **常见误区**：标准过于模糊或复杂；将知名度、财富量或粉丝数等同于认知密度或信息纯度。

2.  **扫描、识别与初步归类**
    *   **输入**：步骤1定义的质量标准，以及自己关注的领域列表。
    *   **操作**：针对每个领域，广泛搜索并列出潜在信息来源（网站、数据库、期刊、人物、平台等）。
        *   依据“信息纯度层级”标准，将信息来源初步归类到对应的层级（A/B/C）。
        *   依据“个体认知密度特征”标准，识别并筛选出该领域内1-3位最符合标准的“高认知密度”个体作为重点跟进对象。
            *   **IF** 找不到完全符合所有特征的人，**THEN** 优先选择在“先舍后取”（行为密度）特征上表现突出者。
    *   **判断标准**：是否在每个关键领域都识别出了不同纯度层级的信息来源（特别是A级来源），以及符合认知密度标准的重点个体。
    *   **常见误区**：只收集中低纯度（B/C级）的便捷来源；选择过多目标个体导致精力分散；因某人某一次观点正确而忽略其整体认知模式的不稳定。

3.  **验证、标注与制定跟进计划**
    *   **输入**：步骤2生成的初步分类列表及重点个体名单。
    *   **操作**：
        *   **对信息来源进行验证和属性标注**：
            *   **更新频率**：实时/日更/月更/年更。
            *   **获取成本**：免费/付费/需要特定资格。
            *   **覆盖范围**：宏观/微观/特定细分领域。
            *   **形式**：数据/分析/观点/混合。
        *   **为重点个体制定具体的“追随”行动计划**：
            *   **IF** 目标个体乐于分享且表达清晰（“福音”），**THEN** 优先选择其系统化输出的内容（如课程、系列文章、著作）进行深度学习。
            *   **IF** 无法直接获取系统内容，**THEN** 采取“案例逆向工程”模式，收集其分散的决策或观点实例，自行归纳背后的模型。
            *   计划需包含具体渠道（如订阅专栏、精读代表作、分析历史案例）和固定的每周投入时间（如X小时）。
    *   **判断标准**：每个重要来源是否有关键属性说明；跟进计划是否具体、可执行。
    *   **常见误区**：只记录来源名称，不记录关键属性；停留在被动浏览或碎片化阅读，缺乏深度思考和结构化整理。

4.  **构建可访问的信息源地图与知识体系**
    *   **输入**：经过步骤3验证和标注的信息来源列表及跟进计划。
    *   **操作**：以结构化方式（如表格、Notion数据库、思维导图）组织信息源，并开始执行跟进计划。
        *   **信息源地图**至少包含字段：领域、来源名称/个体、纯度层级、关键属性、访问链接。
        *   **在吸收高认知密度个体输出时，进行内化与验证**：
            *   能够用自己的话复述其核心观点或模型。
            *   设计一个微小的行动在自己的相关场景中尝试应用，获得实践反馈。
            *   思考引导性问题（如“他做出XX取舍背后的标准是什么？”），进行批判性吸收。
    *   **判断标准**：构建的地图是否易于浏览、搜索和更新；能否从跟进计划中获得经过自身理解修正后的认知点及实践反馈。
    *   **常见误区**：构建后不再维护，导致信息源过时；形式过于复杂；全盘接受失去批判性思维，或急于求成进行大规模实践。

# 输出格式要求

输出应包含两部分：

**第一部分：高纯度信息源地图**
建议采用表格形式，包含以下列：
- **领域**
- **来源名称/个体**
- **类型**（信息来源 / 高认知密度个体）
- **纯度层级 (A/B/C)** 或 **认知密度特征**
- **关键属性**（如：实时数据、深度分析、付费报告、更新频率等）
- **访问方式/链接**（可选）
- **备注/跟进计划**（如：使用技巧、每周投入时间、精读内容等）

**第二部分：个性化建议摘要**
向用户清晰说明：
1.  根据您提供的目标【用户目标】，定义的核心质量特征是：【特征1、特征2】。
2.  基于这些特征，在该领域内值得重点关注的对象有：【对象A（原因）、对象B（原因）】。
3.  建议采取的具体“追随”与内化方式为：【方式1、方式2】，每周建议投入约【X】小时。
4.  在吸收过程中，请特别注意思考：【一个引导性问题】，并通过小范围实践来验证。

[CRITIC]
- 步骤2中“IF 找不到完全符合所有特征的人，THEN 优先选择在‘先舍后取’特征上表现突出者”这一分支，其判断依据（何为“表现突出”）略显主观，缺乏更细化的操作指引。
- 整体流程缺乏明确的错误处理或回退方案。例如，当验证发现某个预设的“高纯度”来源或“高认知密度”个体名不副实时，应如何从地图中移除或降级，并补充新的候选者，此闭环未在步骤中体现。
