# minimal-resistance-path

## When to Use

当需要在复杂多层系统（如职业发展、项目推进、个人成长）中，从当前状态（起点）达成一个明确目标（终点）时，发现直接路径受阻或效率低下，调用此模型寻找阻力最小的非直线路径。description

## Core Logic

# 执行步骤

1.  **识别系统介质层**
    *   **输入**：对当前状态（起点）和目标状态（终点）的清晰描述。
    *   **操作**：分析起点与终点之间的环境，识别出构成阻力的主要“介质层”。这些介质层通常是资源、能力、资格、关系、信息差或环境规则等。将它们列举出来。
    *   **判断标准**：成功识别出至少两个不同的、对达成目标构成实质性障碍的介质层。
    *   **输出**：一份介质层列表（例如：缺乏学历认证、缺少行业人脉、资金不足、关键技能缺失）。

2.  **评估介质特性与穿透阻力**
    *   **输入**：步骤1输出的介质层列表。
    *   **操作**：针对每个介质层，评估“直接穿透”（即试图一步跨越）所需的成本和成功率。同时，思考是否存在可以“绕行”或“折射”的替代入口或中间状态。
    *   **判断标准**：对每个介质层都评估了直接路径的阻力，并至少为一个高阻力介质层设想了一个可能的替代入口或中间目标。
    *   **输出**：一个包含各介质层阻力评估和潜在替代入口的分析表。

3.  **设计折射路径**
    *   **输入**：步骤2输出的介质层分析表。
    *   **操作**：不以直线连接起点和终点，而是设计一条折线或曲线路径。这条路径应依次连接那些“替代入口”或“中间状态”，确保每一步的移动都是当前条件下阻力相对较小的选择。路径的终点仍是最终目标。
    *   **IF** 无法设计出一条看似可行的折线路径：
        *   **操作**：返回步骤1，重新审视或拆分介质层；或返回步骤2，寻找更多替代入口。
    *   **判断标准**：设计出一条包含至少一个“折射点”（非直接目标点）的连贯路径，且能逻辑上解释每一步移动如何降低总阻力。
    *   **输出**：一份分阶段的行动路线图，标明每个阶段（折射点）的目标和预期成果。

4.  **验证与迭代**
    *   **输入**：步骤3输出的行动路线图。
    *   **操作**：执行路线图的第一或前几个阶段。观察实际阻力与预期是否相符，以及是否出现了新的介质层或机会。
    *   **IF** 执行顺利，接近或到达一个折射点：
        *   **操作**：基于新的位置，重新从步骤1开始，更新对介质层的分析，并调整后续路径。
    *   **IF** 执行受阻，阻力远超预期：
        *   **操作**：暂停执行。返回步骤2，重新评估当前介质层的特性，寻找新的替代入口或调整当前阶段目标。
    *   **判断标准**：根据执行反馈，能够动态更新对“最小阻力路径”的理解和规划。
    *   **输出**：更新后的路径规划或调整后的当前阶段行动计划。

# 输出格式要求

向用户呈现一个清晰的“最小阻力路径”方案，包含：
1.  **核心洞察**：重申“两点之间直线最慢”在用户具体情境下的含义。
2.  **识别出的介质层**：列出分析发现的主要障碍层。
3.  **设计的折射路径**：以流程图或阶段列表形式，展示从起点到终点的非直线规划，并简要说明每个“折射点”的价值（如何降低总阻力）。
4.  **下一步行动建议**：明确指出应立即开始的第一项具体、可执行的任务。
