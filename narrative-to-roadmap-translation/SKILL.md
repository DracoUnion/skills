# narrative-to-roadmap-translation

## When to Use

当拥有一个清晰的个人故事框架后，需要将其转化为具体的未来行动指南时description

## Core Logic

# 执行步骤

1.  **【解构故事要素】**
    *   **输入**：来自“story-framework-design”模型的完整故事框架（起、承、转、合）。
    *   **操作**：将故事框架中的抽象叙事要素，转化为具体的行为或状态要素。例如：
        *   “起”中的核心事件 → 识别**初始条件/挑战**。
        *   “承”中的情感与状态 → 明确**需要克服的内在障碍/旧有模式**。
        *   “转”中的关键行动或认知改变 → 定义**核心决策/策略转变**。
        *   “合”中的现状与意义 → 设定**目标状态/希望产生的影响**。
    *   **判断标准**：是否从每个故事步骤中提取出了至少一个可操作或可衡量的具体要素。
    *   **IF** 要素仍停留在抽象叙事层面 → **THEN** 返回步骤1，进行更具体的转化。
    *   **ELSE** → **THEN** 进入步骤2。

2.  **【逆向推导路径】**
    *   **输入**：步骤1提取出的具体要素（特别是“合”中的目标状态）。
    *   **操作**：从“合”（目标状态）开始，反向推导至“起”（现状）。问自己：“要达到‘合’中的状态，在‘转’的阶段我必须完成什么关键跃迁？为了完成这个跃迁，在‘承’的阶段我需要积累哪些能力或改变哪些心态？而这一切的起点（‘起’）我目前处于什么位置？”
    *   **判断标准**：是否形成一条从“现状”到“目标”的、逻辑连贯的反向推导链条。
    *   **IF** 推导链条断裂或逻辑不成立 → **THEN** 返回步骤2，检查每个环节的必要性。
    *   **ELSE** → **THEN** 进入步骤3。

3.  **【转化为行动阶段】**
    *   **输入**：步骤2得出的反向推导链条。
    *   **操作**：将反向链条正向表述为行动阶段：
        *   **阶段一（对应‘起’与‘承’）**：接纳现状，处理内在障碍，积累基础能力。制定本阶段的具体行动与学习计划。
        *   **阶段二（对应‘转’）**：实施核心策略转变或做出关键决策。明确本阶段的里程碑事件或标志性成果。
        *   **阶段三（对应‘合’）**：巩固成果，扩大影响，进入新的稳定状态。定义本阶段的维护性行动与价值输出方式。
    *   **判断标准**：每个阶段是否有明确的焦点、目标和可执行的行动指向。
    *   **IF** 阶段划分模糊或行动不可执行 → **THEN** 返回步骤3，细化各阶段目标与行动。
    *   **ELSE** → **THEN** 流程结束，获得一个与个人故事同构的未来发展路线图。

# 输出格式要求
输出一个三阶段个人发展路线图，需明确：
1.  **阶段一：奠基与内观**（对应起/承）：核心任务、关键行动（1-3项）。
2.  **阶段二：转折与突破**（对应转）：核心决策/转变、里程碑标志。
3.  **阶段三：巩固与辐射**（对应合）：目标状态、价值输出方式。
