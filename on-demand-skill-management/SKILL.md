# on-demand-skill-management

## When to Use

当面临一个新问题或项目，需要快速构建或调用特定能力时；或在知识过载需要优化个人能力库存时description

## Core Logic

# 执行步骤

1.  **定义核心需求与目标能力**
    *   **输入**：具体的任务、挑战、待解决的问题，或能力发展的对象（个人、团队）。
    *   **操作**：
        *   **问题驱动**：将模糊的学习冲动转化为清晰的问题陈述（“为了解决XX问题”）。
        *   **能力分解**：精确分析解决问题或达成目标所需的具体知识、技能、信息，分解为最小可执行单元。
        *   **实力界定**：针对发展对象，区分“硬实力”（可量化、标准化的专业技能，决定能力下限）与“软实力”（认知、协作等系统能力，决定能力上限）。
    *   **判断标准**：输出一个具体的问题陈述，以及一份针对性的、包含硬实力与软实力的必备能力清单。
    *   **输出**：清晰的问题定义与结构化能力需求清单。

2.  **诊断现状与识别缺口**
    *   **输入**：步骤1输出的能力需求清单，以及个人/对象现有的知识技能储备。
    *   **操作**：
        *   **能力匹配**：将现有能力与需求清单匹配，识别“能力缺口”。
        *   **结构诊断**：分析现有能力是源自统一底层逻辑的“同源整料”，还是来源混杂、可能内耗的“异源拼接”。
        *   **状态评估**：评估当前硬实力的关键短板（决定下限）和软实力的核心瓶颈（制约上限）。
    *   **判断标准**：清晰识别出“做什么”（能力缺口）和“是什么”（能力结构是“整料”还是“拼接”）。
    *   **输出**：能力缺口列表、当前能力结构诊断（整料/拼接）、以及实力短板与瓶颈分析。

3.  **制定获取与升级策略**
    *   **输入**：步骤2输出的诊断结果（缺口列表、结构诊断、实力评估）。
    *   **操作**：
        *   **学习筛选**：对所有潜在学习内容，基于与当前问题的**相关性**进行筛选。**高相关**立即学，**中相关**暂存或简化获取，**低/无相关**放弃。
        *   **缺口填补**：针对“能力缺口”，采用最高效方式（搜索、模仿、请教、工具）定向“下载”，以达到“够用”水平。
        *   **系统构建**：**IF** 能力结构为“异源拼接” -> **THEN** 暂停零散学习，定位能统摄多领域的底层逻辑或“元技能”作为修炼核心。**ELSE** (已有“同源”基础) -> 围绕该核心进行深度修炼与整合。
        *   **同步升级**：根据实力评估制定策略：**IF** 硬实力存在危及下限的短板 -> **THEN** 优先补强硬实力。**ELSE IF** 硬实力达标但软实力平庸 -> **THEN** 硬实力专精 + 软实力启蒙。**ELSE IF** 硬实力扎实，软实力是瓶颈 -> **THEN** 重点突破软实力。**ELSE** (两者均高) -> 寻求融合与相互促进。
    *   **判断标准**：策略需明确资源投入优先级，区分短期“下载”与长期“修炼”，并确保学习与问题强相关。
    *   **输出**：一份包含立即学习清单、缺口下载路径、系统能力建设方向及实力升级重点的综合行动计划。

4.  **执行整合与解决问题**
    *   **输入**：现有能力、临时下载的能力包、系统修炼的核心方向、具体问题。
    *   **操作**：将内外部能力、硬实力与软实力进行整合，应用于解决问题。专注于输出结果，并尝试调用正在修炼的系统化底层逻辑。
    *   **判断标准**：问题得到有效解决或推进。
    *   **输出**：问题解决方案或成果。

5.  **评估、卸载与迭代**
    *   **输入**：问题解决结果、临时下载的能力包、系统能力修炼进展。
    *   **操作**：
        *   **能力库存管理**：评估临时能力的未来复用率。**IF** 复用率低 -> **THEN** 主动“卸载”（仅保留关键笔记或路径）。**ELSE** -> 纳入常规库存。
        *   **系统迭代**：定期（如每季度）重复步骤2和3，根据新环境和新目标，更新能力诊断与升级策略，持续修炼系统化能力。
    *   **判断标准**：能力库存保持精简高效，系统化能力得到持续强化，发展策略与时俱进。
    *   **输出**：更新后的个人能力库存状态与下一阶段的迭代发展计划。

# 输出格式要求

以结构化能力管理仪表盘形式呈现：
1.  **需求与诊断报告**：
    *   **待解决问题**：[具体陈述]
    *   **能力缺口清单**：[明确列出]
    *   **能力结构诊断**：[指出属于“拼接型”或“整料型”，并说明依据]
    *   **实力矩阵与瓶颈**：列出关键硬实力与软实力项目，并明确指出当前的“下限锚点”（硬实力短板）和“上限瓶颈”（软实力瓶颈）。
2.  **综合行动计划**：
    *   **立即学习与下载路径**：为高相关性缺口提供1-2个最高效获取途径。
    *   **系统修炼目标**：清晰描述建议长期修炼的底层逻辑或“元技能”。
    *   **同步升级重点**：根据诊断制定的具体、可执行的硬/软实力发展行动。
3.  **整合应用与后续管理提示**：
    *   **整合提示**：提醒如何将新旧、硬软能力组合应用。
    *   **卸载建议**：建议在问题解决后，哪些临时能力可“卸载”。
    *   **迭代提醒**：强调定期复盘与策略更新的重要性。

[CRITIC] 合并后的步骤3（制定获取与升级策略）操作过于复杂，集成了筛选、下载、构建、升级四个子策略，可能令执行者感到困惑或难以聚焦。建议在实际应用中，根据触发器的不同（如“面临新问题” vs “构建长期能力”），引导用户进入不同的策略分支，而非试图一次性处理所有情况。
