# optimal-return-focus-strategy

## When to Use

在资源有限、竞争激烈的环境中，需要以最高效、最稳妥的方式达成核心目标（如赢得比赛、商业竞争、投资获利），而非追求场面优势或短期高光。description

## Core Logic

# 执行步骤

1.  **【环境诊断与战略支点定位】**
    *   **输入**: 对当前局势的初步认知、自身能力与资源清单。
    *   **操作**:
        *   **环境与约束诊断**: 系统分析外部环境（如地理位置、市场趋势、竞争格局）和内部核心约束（如资源稀缺性、关键短板、依赖关系）。明确“不可改变”的硬约束和“可以塑造”的软条件。
        *   **战略支点定位**: 寻找能将外部环境优势与内部能力（或可构建能力）相结合，并能有效规避或转化核心约束的“战略支点”。这个支点应是一个可聚焦发力的关键点，具备“杠杆效应”。
    *   **判断标准**:
        *   是否清晰地识别出根本性的优势机会点和生存约束。
        *   定位的战略支点是否具备“杠杆效应”，即投入相对有限的资源，可能撬动巨大的外部资源或创造系统性优势。
    *   **输出**: 一份包含关键环境要素、核心约束清单的诊断报告，以及一个明确的、聚焦的战略目标或定位陈述。
    *   **常见误区**: 将短期困难误判为根本约束；目标过于宽泛或多元化，导致力量分散。

2.  **【定义并量化“高复利区域”】**
    *   **输入**: 核心目标、战略支点、自身能力与资源清单、环境/对手分析。
    *   **操作**: 围绕战略支点，识别并清晰界定一个具体的“出击半径”或“高复利区域”。该区域需满足：① 在此区域内行动，成功率最高、消耗最低、对达成核心目标贡献最大；② 区域边界可被客观观察或测量（如特定距离、估值区间、安全边际）。
    *   **判断标准**: 能否用一句话或一个量化标准向团队成员明确解释“什么情况算机会，什么情况不算”。
    *   **输出**: 一个明确的、可操作的“高复利区域”定义。

3.  **【评估胜算与构建基础】**
    *   **输入**: 当前局势信息、敌我力量对比、潜在风险点、“高复利区域”定义。
    *   **操作**:
        *   **评估胜算**: 分析直接发起决定性行动（“求战”）的胜率与潜在损失。
        *   **IF** 评估认为有稳胜把握且风险可控 → 可直接进入步骤5（捕捉并执行机会）。
        *   **ELSE** (胜算不足或风险过高) → 进入“构建防御性基础”阶段。
        *   **构建防御性基础**: 不急于进攻，优先投入资源巩固自身基础。构建难以被对手轻易突破的防御体系或竞争壁垒（如物理工事、核心能力、现金流储备、关键关系维护），使我方处于“不败之地”。
    *   **判断标准**: 是否建立了一个能有效抵御对手常规冲击的稳固基础。
    *   **输出**: “直接行动”或“构建基础”的决策，以及相应的行动计划。
    *   **常见误区**: 将资源消耗在非核心的、华而不实的防御上；或在构建过程中因对手挑衅而贸然出击。

4.  **【执行累积性行动与机会筛选】**
    *   **输入**: 已构建的防御性基础（若适用）、“高复利区域”定义、长期目标、实时环境信息流。
    *   **操作**:
        *   **执行累积性行动**: 在稳固的基础上，执行简单、重复、可累积优势的行动（“打呆仗”）。专注于当下任务，不因未来不确定性而分心，也不因过去得失而动摇（“当时不杂，过往不恋”）。
        *   **机会筛选与放弃**:
            *   持续监控环境，将出现的每个机会与“高复利区域”标准进行比对。
            *   **IF** 机会完全落在“高复利区域”内 → 标记为待捕捉机会。
            *   **ELSE** (机会在区域外) → 深度评估其潜在成本（注意力分散、资源消耗、风险暴露、策略偏离），并基于成本分析，明确做出“不行动”或“主动放弃”的决策。记录放弃理由。
    *   **判断标准**:
        *   优势（如阵地、资源、市场份额）是否得到可测量的、持续的巩固和扩大。
        *   面对区域外诱惑时，能否保持纪律，做出基于长期目标的放弃决策。
    *   **输出**: 优势累积记录、被放弃的“机会陷阱”清单及分析、待捕捉的“区域内机会”列表。
    *   **常见误区**: 行动缺乏焦点，不断变换策略；急于求成；无法对“好机会”说“不”。

5.  **【捕捉时机与合力出击】**
    *   **输入**: 累积的优势、待捕捉的“区域内机会”列表、对手状态的变化（如内部瓦解、重大失误）。
    *   **操作**:
        *   **捕捉时机**: 持续监控，但不过度预测（“未来不迎”）。当出现“区域内机会”或对手因长期消耗出现明显弱点时，识别出这一“时机窗口”。
        *   **合力出击**: 一旦时机出现，立即启动预设的、高效的响应流程，并围绕战略支点整合所有可用资源（内力、外力），确保行动高度聚焦、相互增强，形成“合力”，全力获取该次机会的收益。
    *   **判断标准**:
        *   是否明确识别出低成本高胜率的进攻机会。
        *   响应是否迅速、准确、高效，且各项行动呈现“合力”特征。
    *   **输出**: 对一次机会的成功捕获与收益实现。
    *   **IF** 时机未到或出击后未达预期 → 返回步骤4，继续执行累积性行动。

6.  **【环境策略适配与迭代】**
    *   **输入**: 多次执行步骤5的结果（成功/失败记录）、核心目标的达成进度、对环境的持续观察。
    *   **操作**:
        *   **环境策略适配**:
            *   判断所处环境是“封闭系统”（零和博弈）还是“开放/网络化系统”（合作共赢）。
            *   **IF** “封闭系统” → 强化“捂盘型”策略，保护关键信息与资源，巩固壁垒。
            *   **ELSE** “开放系统” → 采用“分享型”策略，主动分享非核心认知以连接高势能伙伴，共同创造增量价值。
        *   **策略迭代评估**:
            *   **IF** 核心目标稳步达成 → 验证当前“战略支点”与“高复利区域”定义有效，坚持现有策略。
            *   **ELSE IF** 目标达成受阻或失败率过高 → 回顾分析：是“支点”或“区域”定义不当，还是执行不力。根据分析结果，返回步骤1或步骤2进行调整。
    *   **判断标准**: 长期目标达成率是否优于追求“场面优势”的替代方案；策略是否与环境属性相匹配。
    *   **输出**: 策略有效性结论、环境适配策略选择及可能的调整方案。

# 输出格式要求

向用户清晰阐述以下要点：
1.  **核心原则**: 追求“投产比最优”和“先胜而后战”。接受“1:0”式的高效胜利，通过“结硬寨，打呆仗”累积不可逆优势，在绝对优势下出击。
2.  **关键动作**: 明确你的“战略支点”和“出击半径”（高复利区域）。在半径外保持绝对耐心并主动放弃诱惑；在构建稳固基础后，于半径内全力、合力出击。
3.  **环境智慧**: 在封闭系统中善于防守和积累，在开放系统中敢于分享和连接。根据环境属性调整你的资源开放度。
4.  **成功心态**: 诀窍在于承受外界对“保守”或“缺乏观赏性”的批评，用最终结果证明策略的正确性。等待和放弃需要比盲动更大的勇气和纪律。

[CRITIC]
1.  **逻辑闭环缺陷**: 步骤3中，若评估胜算后选择“直接行动”，其后续步骤（如如何行动）的描述不如选择“构建基础”的路径详细，存在分支处理不均衡的问题。直接行动路径应明确指向步骤5，但步骤5的输入假设了“累积的优势”，这与直接行动的上下文可能不匹配。
2.  **错误处理缺失**: 整个框架缺乏对“出击失败”后的明确回退或应急方案。步骤5只提到“未达预期则返回步骤4”，但未考虑失败可能造成的资源损失、士气影响或对手反击等具体情况的应对措施。
3.  **术语一致性**: 框架融合了多个来源的术语（如“高复利区域”、“战略支点”、“捂盘型/分享型”），在最终输出给用户时，需要确保用户能理解这些术语的内在联系，避免混淆。
