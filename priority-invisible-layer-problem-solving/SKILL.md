# priority-invisible-layer-problem-solving

## When to Use

当面对复杂、多维度的系统性问题，需要确定解决优先级时description

## Core Logic

# 执行步骤

1.  **【系统界定与问题结构化】**
    *   **输入**：一个复杂的、多因素的系统性问题或现象。
    *   **操作**：
        *   **A. 明确系统边界与核心功能**：识别并描述构成该系统的关键“元素”（如人、物、组织、信息），以及系统作为一个整体所承担的“功能”或“目的”。区分系统内部与外部环境。
        *   **B. 问题维度分析**：将待解决的复杂问题，分解为至少两个对立的分析维度（如：内在/外在、当下/未来、坚持/享受）。
        *   **C. 识别关键参与者与角色**：列出系统内所有重要的个体、群体或组织，分析其核心功能（如生产者、消费者、分解者、连接者）。
    *   **判断标准**：能够清晰划定系统边界，定义其核心功能，并列出关键元素、分析维度和参与者角色。
    *   **输出**：系统初步结构图、问题分析维度、参与者角色矩阵。

2.  **【动态分析与规律洞察】**
    *   **输入**：步骤1的输出。
    *   **操作**：
        *   **A. 分析元素间的相互作用**：分析并描绘关键元素之间如何相互影响、连接和反馈。重点关注“关系”和“互动模式”，抽象出稳定、重复的规则。
        *   **B. 识别反馈回路与依赖关系**：寻找增强回路（导致增长或衰退的循环）和调节回路（趋向稳定的循环）。绘制功能依赖与资源（信息、物质、能量）流向图。
        *   **C. 评估维度状态**：针对每个分析维度，评估其当前状态是“可见/已解决”（表面的、直接的）还是“不可见/未解决”（潜在的、根本的、系统性的）。
        *   **D. 区分竞争模式**：判断系统中的互动是“同向为竞”（目标一致，通过竞争提升整体）还是“异向为争”（目标冲突，消耗资源）。定位“同生态位”竞争关系。
    *   **判断标准**：能够识别出主导系统行为的关键反馈回路和相互作用模式，清晰评估各问题维度的状态，并明确系统中的协作与竞争关系。
    *   **输出**：包含关键动态反馈的系统运作模型、维度状态评估、资源流向与竞争关系图。

3.  **【方案生成与评估】**
    *   **输入**：步骤2的输出；待决策事项或需干预的问题。
    *   **操作**：
        *   **A. 定义“不败”基线**：分析所有可能导致彻底失败的核心风险，设定明确的“安全边界”或不可触犯的“红线”。
        *   **B. 生成候选方案**：基于对系统的理解，构思可能的解决方案或行动路径。可运用“反向工程”思维，将复杂目标逐层分解至可控单元，再从可控支点构思构建路径。
        *   **C. 初步筛选**：使用“不败”基线筛选方案，排除任何存在触犯红线高风险的选项。
        *   **D. 双维度评估**：对通过初步筛选的方案，在“本质性（议题度）”（触及问题核心的程度、战略价值）和“可操作性（解答质）”（实施难易度、成功概率）两个维度上进行独立评分。
        *   **E. 矩阵定位**：将方案放入以“本质性”为纵轴、“可操作性”为横轴的四象限矩阵中，识别出“优先执行区”（高本质性，高可操作性）、“战略规划区”（高本质性，低可操作性）、“战术执行区”（低本质性，高可操作性）和“避免/放弃区”（低本质性，低可操作性）的方案。
    *   **判断标准**：方案生成逻辑清晰，筛选和评估过程客观，矩阵定位明确。
    *   **输出**：“不败”基线清单、候选方案集、方案评估结果与矩阵定位图。

4.  **【决策、干预与定位】**
    *   **输入**：步骤3的输出（特别是矩阵定位结果）；个人/组织的当前状态。
    *   **操作**：
        *   **A. 制定决策与行动计划**：
            *   **IF** 存在“优先执行区”方案 -> 选择并制定详细实施计划。
            *   **ELSE IF** 存在“战略规划区”方案 -> 选择最具潜力者，制定分阶段提升可操作性的路线图。
            *   **ELSE IF** 仅有“战术执行区”方案 -> 评估短期执行价值或重新构思。
            *   **ELSE** (方案均位于“避免/放弃区”) -> 建议放弃并重新定义问题。
        *   **B. 定位干预杠杆点**：明确计划干预的系统层级（元素层、相互作用层、功能/目标层）。评估干预措施对系统功能改变的潜在有效性。相互作用层和功能层通常是高杠杆点。
        *   **C. 战略克制评估**：对于有能力“硬做”的方案，评估其系统性风险（破坏规则、损害关系、影响长期声誉）。探索“探查与借力”（借助系统规则、第三方）的替代方案。做出“能为而不为”或“可为”的最终决策，权衡短期效率与长期系统成本。
        *   **D. 个人/组织战略定位**：从上级系统视角审视自身，评估与系统核心目标及规则的契合度（贡献度、协同性、准备度）。根据评估结果，制定定位调整策略（如培养敬畏心、重新评估价值、聚焦核心竞争力）。
    *   **判断标准**：决策逻辑清晰，干预点选择有据，克制评估全面，定位分析客观。
    *   **输出**：最终决策及详细行动计划、干预方案设计、战略定位分析报告。

5.  **【执行、学习与迭代】**
    *   **输入**：步骤4输出的行动计划；执行过程中产生的新现象和数据。
    *   **操作**：
        *   **A. 主动管理反馈延迟**：对于复杂/延迟反馈系统，主动引入决策“冷却期”，进行结构化信息处理（收集、分类、筛选），基于长期价值而非短期感受做出行动。
        *   **B. 监测系统反馈**：执行干预后，持续观察系统“功能”是否向预期方向转变。**IF** 未变或反向变化 -> 返回步骤1或2，重新分析系统模型。
        *   **C. 问题驱动学习**：在遇到认知瓶颈时，带着具体问题切入复杂材料，寻求专家解读，交叉验证并整合信息，通过“输出”（讲解、写作）倒逼理解深化，并基于新认知提出更深层问题，进入下一学习循环。
        *   **D. 沟通模式分析**：在协作中，分析表达思路（关注过程/结果，认为可变/固定），归类为积极聚焦、消极聚焦、积极发散或消极发散，并采取相应沟通策略。
    *   **判断标准**：执行过程有监控、有反思，能根据反馈调整，学习过程高效。
    *   **输出**：执行进展报告、学习成果、新的驱动性问题、调整后的策略。

# 输出格式要求

根据具体应用场景，选择以下一种或多种结构化格式输出：

**A. 系统分析报告 (适用于步骤1-2)**
1.  **系统概述**：边界、核心功能、关键元素。
2.  **动态模型**：关键相互作用、反馈回路、资源流向。
3.  **问题诊断**：分析维度及状态评估（可见层/不可见层）、竞争关系识别。
4.  **初步洞察**：基于系统规律的核心结论。

**B. 决策与方案评估报告 (适用于步骤3-4A)**
1.  **“不败”基线**：不可触犯的原则清单。
2.  **方案评估**：各方案在“本质性”与“可操作性”上的评分及矩阵定位。
3.  **最终决策**：选择的方案及理由。
4.  **行动计划**：具体实施步骤、监控指标（尤其关注“不败”红线）。

**C. 干预与定位建议 (适用于步骤4B-4D)**
1.  **干预设计**：目标系统、干预点层级（元素/相互作用/功能）、具体措施、预期效果。
2.  **战略克制分析**：“硬做”方案及系统风险、“借力”替代方案、最终决策与理由。
3.  **战略定位分析**：上级系统视角、系统规则、自身契合度评估（优势/差距/风险）、调整策略建议。

**D. 学习与迭代记录 (适用于步骤5)**
*   本轮驱动问题、找到的信息碎片、专家解读、整合笔记、输出物、产生的新问题。

[CRITIC] 合并后的框架步骤繁多，虽逻辑连贯，但在实际单一场景应用时可能显得笨重。用户需要根据具体触发条件（如“需要确定解决优先级”或“需要做出关键决策”）自行判断应重点使用框架中的哪几个子步骤，缺乏一个顶层的、基于触发条件的快速路径指引。
