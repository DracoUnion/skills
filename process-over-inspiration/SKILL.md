# process-over-inspiration

## When to Use

当需要建立可重复、高质量输出的工作流程时description

## Core Logic

# 执行步骤

1.  **识别依赖“手感/灵感”的环节**
    *   **输入**：当前工作流程或任务。
    *   **操作**：分析并列出任务中哪些环节的成功主要依赖于不可控的“手感”、“灵感”或“状态”。
    *   **判断标准**：成功识别出至少一个高度依赖主观、偶然性因素的环节。
    *   **输出**：一份依赖“手感/灵感”的环节清单。

2.  **解构环节，定义“工序”**
    *   **输入**：上一步输出的环节清单。
    *   **操作**：针对清单中的每个环节，将其分解为一系列可重复、可描述的标准化步骤或要素（即“工序”）。例如，将“写出好段落”分解为“确定核心观点 -> 选择论证结构 -> 填充论据 -> 优化措辞”。
    *   **判断标准**：每个被解构的环节都能被描述为3个以上的具体、可操作的子步骤。
    *   **输出**：一份包含关键环节及其对应标准化“工序”的映射表。

3.  **建立并固化标准流程**
    *   **输入**：上一步输出的“工序”映射表。
    *   **操作**：将映射表中的“工序”整合到整体工作流程中，形成新的、不依赖“手感”的标准操作程序（SOP）。
    *   **判断标准**：新的流程文档清晰，任何具备基础能力的人按此流程操作，都能产出质量达标（即使不是顶尖）的结果。
    *   **输出**：一份修订后的、包含明确工序步骤的标准化工作流程文档。

4.  **迭代优化工序**
    *   **输入**：标准化工作流程文档及其实践结果。
    *   **操作**：在多次实践中，收集流程各环节的产出数据和反馈，分析瓶颈或低效步骤，对“工序”进行微调、增删或重新排序。
    *   **判断标准**：流程的稳定性和平均产出质量（而非单次最高质量）得到可衡量的提升。
    *   **输出**：优化后的新版工作流程文档。
    *   **IF** 流程已稳定，产出质量符合预期 **THEN** 进入维护状态，定期审查。
    *   **ELSE** (流程不稳定或质量不达标) **THEN** 返回步骤2，重新解构问题环节。

# 输出格式要求

向用户呈现一个清晰的、可执行的“流程化”方案，包含：
1.  **核心原则**：强调从依赖不可控的“灵感”转向依赖可控的“工序”。
2.  **当前依赖点分析**：指出用户任务中可能过度依赖“手感”的环节。
3.  **具体工序建议**：为上述环节提供初步的、可操作的步骤分解建议。
4.  **下一步行动**：建议用户如何开始建立并记录自己的标准流程。
