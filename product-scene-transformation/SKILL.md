# product-scene-transformation

## When to Use

当需要将传统产品（如书籍）转化为体验式场景，以提升用户参与度和价值感知时description

## Core Logic

# 执行步骤

1.  **识别产品交付的“半成品”状态**
    *   **输入**: 待转化的产品或服务。
    *   **操作**: 分析当前产品交付给用户的形态。判断其是否只是提供了原材料（如信息、功能），而将“组装”、“理解”或“内化”的复杂工作留给了用户。
    *   **判断标准**: 能清晰描述出用户获得产品后，为达成最终目标（如知识应用、技能掌握）仍需自行完成的额外工作。
    *   **输出**: 对“半成品”状态的明确描述。
    *   **IF** 产品交付形态已为用户提供了完整的、低认知负荷的体验闭环，**THEN** 此模型可能不适用，或直接进入步骤4进行优化。

2.  **定义目标“场景”与核心体验**
    *   **输入**: 步骤1的输出（“半成品”描述）。
    *   **操作**: 构思一个能承载产品核心价值的、具体的、高体验感的场景（如“电影院首映式”、“健身私教课”、“探险旅程”）。该场景应具备低进入门槛、高情感卷入和清晰的体验流程。
    *   **判断标准**: 场景描述能让用户立刻理解“我将获得何种感受”，而非“我将得到什么东西”。
    *   **输出**: 目标场景的清晰定义及希望用户获得的核心体验（如“轻松愉悦、心潮澎湃、获得即时效用”）。
    *   **常见误区**: 场景过于抽象（如“学习平台”）或与产品价值脱节。

3.  **定义“真实效用”的行为指标**
    *   **输入**: 产品/服务的核心价值主张、步骤2的输出（核心体验）。
    *   **操作**: 将抽象的价值（如“增长认知”、“学会技能”）或核心体验转化为可观察、可验证的用户行为。例如，“分享内容”、“应用方法解决问题”、“完成某个特定任务”。
    *   **判断标准**: 指标必须是用户外显的、可记录的行为，而非内在感受或意图。
    *   **输出**: 1个或多个用于验证真实效用的关键行为指标。
    *   **IF** 无法定义出明确的行为指标，**THEN** 需要回到价值主张或场景定义阶段，重新澄清产品究竟解决什么问题或提供何种体验。

4.  **设计“场景化”的互动与价值验证机制**
    *   **输入**: 步骤2的输出（目标场景与核心体验）、步骤3的输出（行为指标）。
    *   **操作**: 设计一套简单的规则或仪式，将用户从“被动接收者”转变为“主动参与者”，并内置价值验证环节。规则应服务于筛选真正有需求的用户或强化价值感知。
    *   **判断标准**: 规则能自然引导用户进行自我筛选（是否真需要）或行为反馈（是否真有用），且过程简单、有趣、低压力。
    *   **输出**: 一套可执行的互动与验证规则设计方案。
    *   **示例规则**:
        *   **反向激励筛选**: 向用户提供有吸引力的补偿，条件是用户**承诺不触发**步骤3定义的关键行为指标（例如：保证不阅读、保证不分享）。旨在过滤“非目标”用户。
        *   **高倍补偿承诺**: 向用户提供远高于产品价格的补偿（如10倍退款），条件是用户在使用后，能证明自己**未能达成**步骤3定义的关键行为指标所代表的效用（例如：尝试应用但毫无收获，且能提供简单证明）。旨在建立信任并倒逼价值交付。
        *   **规则需建立在信任基础上，无需复杂监督**。

5.  **重构产品载体与叙事方式**
    *   **输入**: 步骤2的输出（核心体验）、步骤4的输出（互动与验证规则）。
    *   **操作**: 根据目标场景，重新设计产品的表现形式、内容结构和叙事逻辑。使其符合目标场景的“语言”和“质感”（如将知识要点转化为电影剧本式的故事章节）。
    *   **判断标准**: 产品的最终形态能无缝融入步骤2定义的目标场景，用户接触时产生的是场景预期的体验，而非处理“半成品”的负担。
    *   **输出**: 新的产品载体设计方案或内容框架。
    *   **常见误区**: 只改变包装文案，未触及内容结构和体验流程的本质。

6.  **实施并基于反馈迭代**
    *   **输入**: 步骤4的输出（互动与验证规则）、实际用户参与数据。
    *   **操作**: 运行该场景化产品及验证框架。观察：1) 有多少用户接受了“反向激励”；2) 有多少用户触发了“高倍补偿”条款及其原因；3) 用户对场景化体验的整体反馈。
    *   **判断标准**: 能清晰区分出“高价值用户”、“低需求用户”以及“产品价值交付或场景体验的薄弱环节”。
    *   **输出**: 关于用户真实需求强度与体验满意度的洞察，以及产品与场景需要改进的具体方向。
    *   **IF** 大量用户接受反向激励，**THEN** 说明市场定位、价值传播或场景吸引力可能有问题。
    *   **IF** 大量用户触发高倍补偿，**THEN** 说明产品价值交付或场景体验存在缺陷，需根据反馈原因进行迭代。

# 输出格式要求

输出应包含：
1.  **当前产品“半成品”状态分析**。
2.  **建议的转化目标场景**及其希望带来的核心体验。
3.  **为验证产品价值所定义的关键用户行为指标**。
4.  **具体的场景化互动与价值验证规则设计**（包含反向激励与高倍补偿机制）。
5.  **对产品载体/内容的重构方向建议**。
6.  **对可能出现的反馈结果的预分析及对应的迭代方向**。

[CRITIC]
- 步骤3（定义行为指标）的前置条件中，“核心体验”作为输入，其与“行为指标”的转化逻辑和判断标准未详细说明，可能存在操作模糊性。
- 步骤4合并了原有两个Skill的规则设计，但未明确说明在何种情况下优先使用“反向激励”还是“高倍补偿”，或两者是否必须同时部署，存在策略选择缺口。
- 整体流程缺乏明确的“回退方案”。例如，若在步骤6中发现场景转化完全失败（如无人参与），应如何退回至传统产品形态或重新定义场景，流程中未作说明。
