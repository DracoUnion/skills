# raw-information-extraction

## When to Use

当需要获取高保真、未经加工的一手信息以支持关键决策时description

## Core Logic

# 执行步骤

1.  **明确目标与提取核心命题**
    *   **输入**：具体的信息需求、决策问题或待验证的对象（如一段叙述、一个宣称）。
    *   **操作**：
        *   **IF** 目标是获取一手信息 -> 分析问题，列出所有可能产生相关一手信息的源头。
        *   **IF** 目标是评估信息是否被稀释/歪曲 -> 剥离情感渲染，提取信息试图传达的核心观点、结论或定性。
        *   **IF** 目标是验证真伪/可行性 -> 剥离表象，提取其核心事实主张或关键属性。
    *   **判断标准**：能够清晰、中立地陈述信息需求或可被验证的核心命题。
    *   **输出**：一份潜在一手信息源列表 **或** 一个明确的核心主张/命题陈述。

2.  **追溯与获取原始档案/信息源**
    *   **输入**：潜在一手信息源列表 **或** 核心主张/命题。
    *   **操作**：寻找并获取与目标最相关、最接近源头的一手档案或权威信源。源头优先级：事件亲历者/原始陈述 > 原始文档/数据/报告 > 现场记录 > 早期直接报道/权威研究。
        *   记录信息的具体内容、时间、地点、发布者等元数据。
        *   **IF** 无法找到原始来源 -> 标记为“来源不明”，评估终止，结论为“可信度存疑/无法验证”。
        *   **IF** 只有单一信源 -> 标记为“信源单一，需交叉验证”。
        *   **IF** 信源均为二手转述或带有明显立场 -> 标记为“信源可靠性存疑”，并继续追溯更原始档案。
    *   **判断标准**：是否找到了至少一个可追溯、可验证的高相关性原始或权威信源，并对信源链完整性有评估。
    *   **输出**：忠实记录的原始事实摘要（附带来源说明） **或** 与核心命题相关的原始/权威档案集合及质量评估。

3.  **深度比对、验证与纯度评估**
    *   **输入**：
        *   **路径A（提取与评估）**：忠实记录的原始信息摘要。
        *   **路径B（验证与鉴别）**：核心主张/命题、相关原始档案集合。
    *   **操作**：
        *   **路径A（提取与评估）**：审视原始信息。
            *   **IF** 信息包含大量形容词、情绪化语言或无法验证的断言 -> 标记为“已稀释”，需返回步骤2寻找更纯净源头或谨慎使用。
            *   **IF** 信息主要为客观事实、数据或亲历者直接陈述 -> 标记为“毛金信息”，进入步骤4。
        *   **路径B（验证与鉴别）**：进行系统比对。
            *   **完整性/真实性比对**：检查待评估信息与原始资料是否存在：省略关键上下文、夸大/扭曲事实（如可能变必然）、嫁接无关信息。
            *   **交叉验证与逻辑比对**：将档案信息与核心命题比对，标记“初步证实”、“证伪点”或“存疑点”。检查不同档案间矛盾，并用基础原理进行逻辑推理。
            *   **评估动机与影响**：基于信息呈现方式（如标题党、情绪化用语）和比对结果，推断传播可能目的（吸引点击、误导）及潜在影响。
    *   **判断标准**：
        *   （路径A）能否清晰区分信息中的客观事实与主观解读。
        *   （路径B）比对是否系统全面，发现的差异点是否有原始资料证据支持；推断是否合理。
    *   **输出**：
        *   **路径A**：带有“毛金信息”或“已稀释”标记的评估结果。
        *   **路径B**：一份验证分析报告（列出支持点、证伪点、存疑点及逻辑推断）及对传播动机/影响的推断。

4.  **综合判断与提炼产出**
    *   **输入**：
        *   **路径A**：标记为“毛金信息”的评估结果。
        *   **路径B**：验证分析报告及动机影响推断。
    *   **操作**：
        *   **路径A（提炼洞察与边界）**：从“毛金信息”中提炼核心模式、关键数据或需求。明确该信息的边界（具体时间、地点、条件）。注明直接套用的风险。
        *   **路径B（做出最终判定）**：基于证据权重、逻辑自洽性及动机分析做出判断。
            *   **IF** 存在强有力一手证据证伪 -> 结论为“命题很可能不成立”。
            *   **IF** 发现实质性断章取义/歪曲，且动机主要为博眼球/误导 -> 判定为“掺水信息”。
            *   **IF** 证据充分一致支持，且信息基本忠实 -> 结论为“命题在现有证据下成立”或“基本可信信息”，同时注明证据边界。
            *   **IF** 证据严重不足、矛盾无法调和或来源不明 -> 结论为“无法判断/验证信息”，指出关键缺失。
    *   **判断标准**：结论是否有明确证据/逻辑支撑，是否清晰说明了结论的适用范围和不确定性。
    *   **输出**：包含核心洞察与明确边界说明的最终信息报告 **或** 关于核心命题/信息可信度的综合判定结论及置信度说明。

# 输出格式要求

根据初始目标，选择以下一种结构化格式呈现：

**A. 信息提取与洞察报告（当目标是获取一手信息）**
1.  **决策问题/信息需求**：[复述输入的问题]
2.  **信息源头**：[列出查询的一手信息源及其性质]
3.  **原始事实还原**：[按时间或逻辑顺序排列的客观事实与数据]
4.  **信息纯度评估**：[毛金信息/已稀释] 及简要理由
5.  **核心洞察**：[从原始事实中提炼出的关键点]
6.  **应用边界与提示**：[该信息的产生背景与直接套用的风险提示]

**B. 信息可信度评估报告（当目标是评估信息是否被稀释/歪曲）**
1.  **核心主张**：[待评估信息试图传达的主要观点]
2.  **信息完整性评估**：[简述与原始资料的比对发现，如“省略了XX关键前提”、“将‘可能’夸大为‘必然’”]
3.  **可信度判定**：[掺水信息 / 基本可信信息 / 无法验证信息]
4.  **处理建议**：[根据判定给出具体行动建议]

**C. 信息真伪/可行性验证报告（当目标是验证某个命题）**
1.  **待验证的核心命题**。
2.  **查阅的关键档案/信源列表及质量简述**。
3.  **详细的比对分析过程**（支持点、矛盾点、存疑点及逻辑推断）。
4.  **最终结论、置信度及适用范围**。

[CRITIC] 合并后的步骤3（深度比对、验证与纯度评估）中，操作部分同时描述了**路径A**和**路径B**，但**判断标准**和**输出**部分的结构暗示用户需要根据初始目标选择一条路径执行。然而，步骤逻辑上未强制要求或明确引导用户在步骤1之后就必须严格区分路径并仅执行其一。在实际操作中，用户可能同时有“获取一手信息”和“验证其纯度”的需求，这会导致路径A和B的部分操作重叠但输出目标不同。当前的合并步骤试图涵盖三种场景，但未彻底解决当用户需求复合时（例如，既想提取毛金信息，又想验证其是否被掺水），步骤执行的清晰度和输出归并的逻辑。建议明确：若需求复合，应优先执行溯源与比对（路径B的核心操作），再基于纯净信息进行提炼（路径A的后半部分），并相应调整输出格式为复合报告。
