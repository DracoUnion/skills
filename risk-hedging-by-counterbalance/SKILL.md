# risk-hedging-by-counterbalance

## When to Use

当主营业务或核心资产面临明确的、难以避免的系统性外部风险（如原材料价格波动、技术颠覆、政策变化）时，需要设计保护性策略。description

## Core Logic

# 执行步骤

1.  **识别核心风险暴露**
    *   **输入**：主营业务或核心资产描述。
    *   **操作**：分析并确定对其盈利或生存构成最大威胁的**外部系统性风险**（通常为价格风险、需求风险、技术替代风险等）。
    *   **判断标准**：风险因素明确、可衡量，且对业务有直接的负面影响。
    *   **常见误区**：将内部管理问题或运营效率低下误判为需要对冲的外部风险。

2.  **寻找对冲标的**
    *   **输入**：步骤1中识别的核心风险。
    *   **操作**：寻找与核心风险呈**负相关**或**弱相关**的资产、业务或投资机会。标的应满足：
        *   **负相关逻辑**：当核心风险发生时，对冲标的的价值上升或业务受益。
        *   **可操作性**：能够通过投资、开展新业务、建立合作伙伴关系等方式实际持有或接触该标的。
    *   **判断标准**：能清晰阐述对冲标的与核心风险之间的制衡逻辑。
    *   **IF** 找不到符合条件的对冲标的 -> 考虑其他风险管理手段（如保险、多元化），本模型不适用。
    *   **ELSE** (找到对冲标的) -> 进入步骤3。

3.  **构建对冲组合**
    *   **输入**：核心业务与找到的对冲标的。
    *   **操作**：确定对冲的比例和方式。原则是使**整体组合**的风险降低，而非追求对冲标的本身利润最大化。
        *   **完全对冲**：旨在完全抵消风险，稳定收益（如期货套保）。
        *   **部分对冲**：旨在降低风险敞口，同时保留部分上行空间。
    *   **判断标准**：能够说明在不同风险情景下（风险发生/不发生），整体组合的损益情况比不对冲时更稳定或更优。
    *   **常见误区**：对冲头寸过大，变成以投机对冲标的为主业，本末倒置；或对冲成本过高，侵蚀主业利润。

4.  **执行与监控**
    *   **输入**：步骤3设计的对冲组合方案。
    *   **操作**：
        *   执行对冲操作（如进行投资、开展新业务线）。
        *   定期监控核心风险与对冲标的之间的关系是否发生变化。
    *   **判断标准**：对冲操作已实际执行，并建立了监控机制。
    *   **常见误区**：“设而不管”，当市场基础逻辑变化时未能及时调整对冲策略。

# 输出格式要求
输出应包含：
1.  **核心风险**：明确描述需要对冲的风险。
2.  **对冲逻辑**：解释选择的对冲标的为何能与核心风险形成制衡。
3.  **对冲方案**：说明具体的对冲方式、比例或投入。
4.  **预期效果**：描述对冲后，在不同市场情况下整体资产或业务的损益变化。
