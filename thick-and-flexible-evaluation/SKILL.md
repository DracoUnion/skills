# thick-and-flexible-evaluation

## When to Use

当需要评估一个候选人或合作伙伴的内在品质，尤其是在信息不对称或缺乏客观标准的情况下description

## Core Logic

# 执行步骤

1.  **设计评估问题**
    *   **输入**：评估对象的背景信息（如简历、专业领域）。
    *   **操作**：基于其背景，设计一个关于其专业领域基础概念的定义性问题。该问题需满足：
        *   有明确、公开、可查证的标准答案（如教科书、百科定义）。
        *   属于基础知识，但有一定概率被遗忘或记忆模糊。
    *   **判断标准**：问题设计完成，且满足上述两个条件。
    *   **输出**：一个具体的评估问题。

2.  **提出评估问题并观察反应**
    *   **输入**：步骤1设计的评估问题。
    *   **操作**：向评估对象清晰提出该问题，并观察其即时反应和回答内容。
    *   **判断标准**：问题已提出，并获得了评估对象的回应。
    *   **输出**：评估对象的回答内容及态度。

3.  **根据反应模式进行分类与评估**
    *   **输入**：步骤2获得的回答内容及态度。
    *   **操作**：将回答与以下模式进行匹配，并做出初步评估：
        *   **IF** 回答是“一字不差复述标准答案” → **THEN** 归类为“A类：知识扎实但灵活性待定”。评估：需结合其他表现综合判断，不直接否定。
        *   **IF** 回答是“胡诌一个错误答案” → **THEN** 归类为“B类：不忠于事实/不厚道”。评估：存在诚信风险，建议过滤。
        *   **IF** 回答是“承认忘记或不知道” → **THEN** 归类为“C类：厚道但可能缺乏主动性”。评估：诚信可靠，但创新或问题解决能力可能有限，属于安全选项。
        *   **IF** 回答是“承认不知道并主动请求查询或学习机会” → **THEN** 归类为“D类：厚道且灵活”。评估：兼具诚实与主动解决问题的灵活性，为高价值人选。
    *   **判断标准**：成功将对象反应归入上述四类之一，并得出相应的初步评估结论。
    *   **输出**：对评估对象的“厚道”与“灵活”两个维度的定性判断及综合评估等级（A/B/C/D）。

# 输出格式要求
以结构化方式呈现评估结果：
- **评估问题**：[重复提出的问题]
- **对象反应**：[简述对象的回答和态度]
- **归类结果**：[A/B/C/D类]
- **品质分析**：
  - 厚道度：[高/中/低，基于是否忠于事实]
  - 灵活度：[高/中/低，基于是否主动寻求解决方案]
- **综合评估与建议**：[基于分类的简要结论与后续行动建议]
